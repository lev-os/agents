================================================================================
KEY INSIGHT: THE RESEARCH HUB PROBLEM
================================================================================

PAIN POINT:
  Research processes (multi-source web scraping, document analysis, 
  multi-turn LLM queries) are opaque to users:
  - No visibility into which sources are queried
  - No confidence feedback as research progresses
  - No cost transparency until job completion
  - Terminal filled with raw logs that obscure actionable status

TARGET OUTCOME:
  A real-time CLI tool that shows:
  ✓ Active research queries with confidence levels (0-100%)
  ✓ Source discovery progress (queued → running → fetched → parsed)
  ✓ Cost tracking (API calls, tokens, $/question)
  ✓ Multi-turn research status (Q→A chains visible)
  ✓ Interactive controls (pause/resume, adjust scope, inline review)

================================================================================
ARCHITECTURE DECISION: INLINE MODE + ELM/BUBBLETEA
================================================================================

WHY INLINE MODE (not alt-screen)?
  ✓ Preserves scrollback logs for audit & debugging
  ✓ No full-screen flicker or cursor drift
  ✓ Multi-terminal friendly (parallel research sessions)
  ✓ Uses FrankenTUI's hybrid strategy: scroll-region optimization + 
    overlay redraw fallback

WHY ELM/BUBBLETEA PATTERN?
  ✓ Deterministic state (confidence, cost, counts from single source)
  ✓ Message-driven (backend sends events; UI reacts)
  ✓ Testable via snapshot tests
  ✓ Hot-reload ready for rapid iteration

WHY GRACEFUL DEGRADATION (BUDGET SYSTEM)?
  ✓ 1000-cell terminal: render all 5 zones
  ✓ 500-cell terminal: omit cost tracker, compact log
  ✓ 200-cell terminal: header + progress + footer only
  ✓ Under load (frame > 16ms): essential widgets always render

INTERACTION MODEL (KEYBOARD SHORTCUTS)
================================================================================

CORE ACTIONS:
  SPC              → Pause/Resume research
  A                → Open scope adjustment modal
  + or Esc Esc     → Toggle sources overlay
  Q or Ctrl+C      → Quit (gracefully or hard)
  Ctrl+D           → Soft quit (cancel if running, else exit)

NAVIGATION:
  PageUp/Dn        → Scroll research log
  ↑/↓ (overlay)    → Navigate sources
  Enter (overlay)  → Expand/collapse excerpt

COPY/BROWSE:
  C (overlay)      → Copy URL to clipboard
  O (overlay)      → Open in browser

UTILITY:
  Ctrl+T           → Cycle theme (dark/light/alt)

STATE MACHINE:
  IDLE → RUNNING → PAUSED → COMPLETED
       ↓ (quit)           ↓ (quit)
      EXIT               EXIT

================================================================================
DESIGN PRINCIPLES
================================================================================

1. TRANSPARENCY
   - Real-time confidence feedback builds trust
   - Cost visibility prevents surprises
   - Source list enables audit & reproducibility

2. RESPONSIVENESS
   - Keyboard shortcuts < 50ms latency
   - Pause/resume without data loss
   - Interactive scope adjustment mid-research

3. PRESERVATION
   - Scrollback logs never truncated
   - Session metadata saved for later review
   - Research history accessible

4. GRACEFUL DEGRADATION
   - Essential widgets always render
   - Non-essential omitted under load
   - Works on 80x24 through 160x50 terminals

5. STREAMING-FIRST
   - Backend sends events; UI reacts
   - No blocking I/O in render loop
   - Handles continuous source discovery
