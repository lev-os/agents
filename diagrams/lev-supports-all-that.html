Nested Loops — The Hopf Intuition Made Concrete
01-02 Nested Hopf Tori → Lev's Three Loop Scales
Location + Phase + Larger Cycle
The Hopf intuition says a system has a base state, a local phase within a loop, and a larger-scale nesting of loops. Lev makes this literal with three nested execution levels, each with its own cycle rate and exit logic.
Loop ScaleMechanismCycle RateExit Conditions
Outer (heartbeat) runHeartbeat() — generic sleep-paced FSM with pluggable callbacks Adaptive: speeds up 0.5× when work found, slows down 2.0× when idle, clamped [60s, 15m] 9 types: no_work, budget_exhausted, max_ticks, until_met, all_done, fail_fast, interrupted, circuit_breaker, error
Middle (orchestration) lev loop — discover → validate → promote → budget per entity Per-entity within each heartbeat tick Entity blocked, budget exceeded, no entities ready
Inner (Tick) INGEST → OBSERVE → PROPOSE → GATE → APPLY → UPDATE → EMIT Every message / event Gate rejection, semantic completion detection, context pressure
The inner loop also has its own nested execution: executeIterativeLoop() runs semantic condition checking with retry/budget/idle detection. It emits lifecycle events — iteration.started, iteration.completed, loop.idle, loop.exit — so every loop level is observable.

Topology / Orchestration / Dispatch
03-04 The Clean Three-Way Split + Phase
Lev's Four-Plane Boundary Is This Idea, Implemented
Topology = FlowMind declarations (.flow.yaml) define the graph of what can connect to what. You change the shape without touching execution. FlowMind compiles to 6 targets: smartdown, system prompt, prompt gen, hooks, schedule, context assembly.
Orchestration = core/orchestration/ manages movement through the structure. DAG scheduling uses Kahn's topological sort with cycle detection. Worker coordination, entity queue promotion, budget management. It executes policy-scoped intents — it does not own policy truth.
Dispatch = core/exec/ via createExec(dispatch). Returns a frozen ExecEngine with per-provider concurrency (ProviderSemaphore with backpressure + queueing). No module-level mutable state. Provider-agnostic — swap transport without touching topology.
Phase (the Hopf extra) = core/event-machines/ tracks which phase each entity is in. The entity lifecycle FSM: ephemeral → captured → classified → crystallizing → crystallized → manifesting → completed. Same graph position means different things depending on phase.
The boundary invariants are explicit: FlowMind does not own worker pool internals. Orchestration does not own policy truth. Graph does not schedule execution. Cross-plane boundaries must emit canonical LevEvent with correlation IDs.

Structured Disagreement, Not Consensus
The CDO (Command-Driven Operations) system implements multi-view deliberation with five explicit roles, each with different emphasis layers:

State & Update Discipline
11-12 Bayesian → Event-Sourced State
State-via-Event, Not State-via-Mutation
All inter-module state changes flow through LevEvent — a canonical event shape with version, id, type, source, time, data. 67+ typed event types across exec, daemon, supervisor, worker, and task domains. State is never mutated directly across boundaries.
The SessionAggregate implements full event-sourced state reconstruction: snapshot (compressed baseline) + event tail (full replay from snapshot). Every state can be reconstructed at any point in time. TRANSITIONS state machine enforces valid state→command paths. Uncommitted events are buffered until snapshot, then cleared.
This is the Bayesian update discipline stripped of probability: you always know what changed, when, and why. The "prior" is the snapshot. The "evidence" is the event tail. The "posterior" is the reconstructed state.

14 FEP → The Tick IS Active Inference
PROPOSE → GATE → APPLY → UPDATE
The Tick loop is a prediction-mismatch-update cycle without the variational baggage:
OBSERVE = gather current state. PROPOSE = predict what should happen. GATE = check proposal against constraints (the mismatch). APPLY = if it passes, execute (the update). UPDATE = write to graph. EMIT = broadcast event.
The semantic completion detector (detectSemanticCompletion()) is another mismatch signal — a two-pass regex matcher that detects when the system's output indicates the loop condition is satisfied. Context pressure calculation provides yet another: when token budget is exhausted, the system must exit.

Structured Output & Guards
19-21 Constrained Decoding + Guardrails + Typed Synthesis
Schema-Driven Output + Multi-Stage Gate Pipeline
Constrained decoding: FlowMind schemas constrain what agents can produce. OutputConfig specifies format (JSON/YAML/Markdown/text), schema, and template per node. The codegen pipeline generates typed surfaces (Zod schemas) from service manifests. Output shape is declared, not hoped for.
Guardrail pipeline: The GATE step in every Tick is the guardrail. Is the proposal well-formed? Does it pass policy constraints? Is the move legal in this state? Sensitive operations trigger AgentLease step-up auth. Nothing proceeds without gate clearance. Safety config includes allow-list, deny-list, require_approval, timeout_ms, max_retries.
Typed synthesis: Plugins declare capabilities via leviathan key in package.json + config.yaml. The plugin validator (@lev-os/poly/plugin-validator) checks contracts at registration. Candidates that don't satisfy the contract are rejected. compareFingerprints() validates determinism.
The full gate pipeline: INGEST → (Gate 1: Planning/DoR) → (Gate 2: Skill discovery) → PROPOSE → (Gate 3: Pre-exec validation) → APPLY → EXECUTE → (Gate 4: Post-exec DoD) → EMIT.

Search & Graph
28 Graph Mining → Universal Context Graph
Everything is a Graph (Principle #7)
core/graph/ implements a universal context graph. Entity types: spec, task, proposal, session, commit, agent, intent, episode. Properties stored as Claims (kind, predicate, object, truth_state). Relationships stored as Links (source → target with type and weight 0-1). Supporting data as Evidence (kind, source, payload_ref, reliability score).
Graph compositor pipeline: Event → Extract (adapters fan out) → Classify (FlowMind policy slot) → Validate (hooks) → Apply (all backends). Topology types include DAG (Kahn's sort), semantic graph (weighted PPR), and lineage graph (causal parent tracking).

23, 26 Abstract Interpretation + Info Geometry → Graph Depth
Cheap First, Deep Later. Distance is Meaningful.
Entity lifecycle goes ephemeral → classified → crystallized → proposal → execution. This is abstract interpretation: cheap rough pass first (classify), deeper analysis only for what survives (crystallize), expensive execution only for gated proposals.
The graph's depth levels (L0 snapshot → L1 → L2 → L3 full context) implement information geometry: entities at different depths have different distinguishability. An L0 entity is a sketch; an L3 entity is fully characterized. The "distance" between states is meaningful, not coordinate-based.

31 AlphaGeometry → lev loop
Disciplined Search with Aggressive Pruning
lev loop is disciplined search: discover entities → validate → promote what survives → budget resources. No search explosion. The heartbeat's adaptive interval control (speed up 0.5× for work, slow down 2.0× for idle, clamped to [60s, 15m]) prevents both wasted cycles and missed opportunities. Circuit breaker trips after 3 consecutive failures. Partial progress is cached in the graph — entities don't lose state between iterations.
The durable task queue (core/orchestration/src/queue/durable-task-queue.ts) persists to JSONL with crash recovery and cooperative cancellation. Search state survives restarts.
