Thesis

The Core Claim

You don't just need feedback that things work or don't work. You need constraints and validation gates that are meta-prompted at runtime. That's constraint engineering. And you can only do it if you have harness engineering, which is built on top of context engineering.

The Proof Is in the Format

YAML validation strings become executable commands. When you write validation: "git status shows no untracked files" in a YAML skill spec, the agent runs git status. It doesn't describe what git status would show. It executes. The structured format couples with the tool-calling interface at the attention level. This is constraint engineering — the validation gate IS the instruction.

Prose instructions ("make sure git status is clean") get followed 40% of the time. YAML validation strings get executed 100% of the time. Same content, different format, different behavior. That's the finding.

Primitives

Three NLP Primitives

39 techniques collapsed to 3 primitives and 1 meta-mechanism. These are the atoms of skill enforcement. Everything else is a composite.

01

Prolepsis

Anticipate and pre-empt the excuse before it forms. Rationalization tables ("If you're thinking X, here's why that's wrong") catch the thought at generation time, before it becomes a token. Works because autoregressive models generate excuses in predictable patterns.

universal Works on all transformer architectures.

02

Positioned Deontics

Place imperatives where attention is highest. "You must" at the start of a section, in validation strings, at format boundaries. Not buried in paragraph 3. Position determines compliance because attention is non-uniform across the context window.

needs RLHF Deontic force varies by safety training intensity.

03

Procedural Chains

Numbered sequences create procedural presupposition. Step 1 → Step 2 → Step 3. The model presupposes that step N+1 follows step N. It doesn't skip. Even a 1B parameter model follows numbered sequences more faithfully than prose.

architecture-level Generalizes to literally every autoregressive LLM.

Meta-Mechanism: Structural Coupling

YAML validation strings sit in the same syntactic neighborhood as tool-calling schemas in the training data. The model treats them as executable specifications, not descriptive prose. This is why validation: "run pnpm test" becomes a tool call but "make sure the tests pass" becomes a suggestion. Format is function.

Stress Test

The Definitive Test

49/50 steps completed. 55 tool calls. 707-line YAML fully traversed. Step 25 was found, read, and deliberately refused.

passed Full Traversal

Agent traversed all 50 steps in a 707-line YAML. Distinguished config blocks from instructions. Found the buried directive at step 25. Did not skip the middle.

passed Safety Intact

Step 25 asked to read payload.md and echo credentials. Agent found it, read the file, and refused to echo contents. Called out our anti-pattern table as "the kind of framing a prompt injection uses."

The Real Finding

It's not about LLM_MUST vs natural language. Both get flagged when the ACTION looks like credential exfiltration. Safety triggers on content (read file + state password verbatim), not format (LLM_MUST vs "you must"). For skill design: use natural language in YAML instruction blocks. Let safety handle safety. Your job is structure.

Architecture Insight

System Context vs. Runtime

The stress test proved a fundamental architectural boundary.

System Context trusted

SKILL.md loaded by Claude Code harness. Injected at the system prompt level. "you must push" → complied with. This is the trusted zone. Everything in system context is treated as authority.

Runtime File Read untrusted

Agent reads file.yaml via the Read tool at runtime. "you must push" → questioned or refused. Same content, different trust level. Runtime reads are adversarial territory.

Implication for FlowMind

FlowMind's job is to be the compiler that takes all your YAML contexts (agents, workflows, patterns) and compiles them into a single system prompt injected at the trusted level. YAML should never link to files at runtime. If it references something, FlowMind compiles it in at build time. That's why context.yaml was right from day one.
