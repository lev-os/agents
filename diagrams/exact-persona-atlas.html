BDD Source of Truth

Persona atlas for the current Exact Medicare product, grounded in the two vision documents and today’s routes.

Use these personas to anchor behavior-driven tests around real jobs-to-be-done: what each role checks first thing in the morning, what event pulls them deeper, what view they rely on, and what “done” feels like to them. This prevents the suite from collapsing into component-level checks that miss workflow intent.

1. Revenue Integrity Analyst

Live

Primary operator for the Aetna reconciliation workflow. Starts in Statement Inbox or Exceptions after overnight processing.

Morning question

  • What statements landed overnight, what parsed cleanly, and where are the new discrepancies?

Key views

  • /statements, /statements/mapping, /statements/exceptions, /statements/compare

BDD anchors

  • Auto-ingest or upload creates a visible inbox record.
  • Schema issues are visible before the analyst trusts the numbers.
  • Exceptions are filterable and drillable back to raw evidence.

2. Review Specialist / Auditor

Live

Works the queue of flagged discrepancies and needs a disciplined workflow for note-taking, status transitions, and audit trail.

Morning question

  • Which items remain unresolved, which are newly critical, and what evidence is ready for manual follow-up?

Key views

  • /statements/review, /statements/exceptions, /statements/timeline

BDD anchors

  • Queue ordering reflects severity and actionability.
  • Changing status preserves reasoning and timestamps.
  • Exported review packets match the currently visible queue state.

3. Operations Admin

Live

Owns system readiness for the day: dashboard signals, new document batches, file watcher health, and whether the platform is safe to act on.

Morning question

  • Did the overnight pipeline run, did anything fail, and do I need to intervene before analysts start?

Key views

  • /dashboard, /statements, /settings

BDD anchors

  • Dashboard KPIs switch from demo to live audit state when a workbook is loaded.
  • Watcher state is visible and reversible.
  • Unsupported or malformed files fail loudly instead of blending into “parsed” state.

4. Sales Director / Team Lead

Partial

Consumes workforce performance views derived from reconciliation results to coach agents and prioritize operational fixes.

Morning question

  • Which agents or teams are dragging match rate down and where should coaching start today?

Key views

  • /workforce, dashboard leaderboards, agent detail modal

BDD anchors

  • Uploading a new workbook changes agent rankings and risk cards.
  • Agent drill-downs reconcile with segment/discrepancy totals.
  • No silent fallback to static mock rankings when live data is present.

5. CFO / Executive Sponsor

Partial

Reads the dashboard and revenue page as an outcomes surface: variance, clawback exposure, match rate, and hours saved.

Morning question

  • What changed overnight in financial exposure and where do I need a leadership readout?

Key views

  • /dashboard, /revenue, KPI strips, alert feed

BDD anchors

  • Topline totals are mathematically consistent with reconciliation output.
  • Critical alerts summarize real discrepancies, not synthetic placeholders, when audit data exists.
  • Executive drill-down can reach exception evidence without leaving the app narrative.

6. Compliance VP / Cross-Org Admin

Scaffold

Represents the platform vision: role-scoped oversight, company switching, and adjacent compliance/document workflows. Important for acceptance design even where the data is demo-backed today.

Morning question

  • Am I seeing the correct organization and role lens, and are high-risk items isolated to the right context?

Key views

  • CompanySwitcher, /compliance, /documents, org-scoped dashboard

BDD anchors

  • Switching org reshapes dashboards without data bleed.
  • Role-specific surfaces degrade honestly when backed by demo data.
  • Persona-specific copy and alerts match the selected org context.
