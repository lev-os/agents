Runtime Flow

The correct data path already exists, but it is Aetna-specific and too permissive.

The live path is StatementInboxCard → useInboxData → AuditContext.uploadFile → parseFile/parseExcelFile → reconcile → audit views. The risk is not missing plumbing. The risk is that unsupported uploads still flow through and some audit surfaces still add synthetic labels or workflow state.

Trade-Off

Candidate approaches

Option A: Aetna-only hard gate Recommended

Block unsupported files, surface parser/schema failures, pin math to the controlled workbook, and remove demo overlays from audit views. Fastest route to deterministic trust.

Option B: Carrier-pluggable runtime Later

Wire carrierInterface end-to-end and introduce per-carrier parsers/rulesets. Correct long-term direction, but too large if the immediate need is trustworthy Aetna audit output.

Option C: Generic schema-first upload Reject now

Broadest intake story, but weakens determinism because mapping ambiguity moves into runtime/user interpretation rather than fail-fast control.

Support Gate Wrong carrier or arbitrary workbook presented as valid audit input. Accept only supported sheet/header contracts. Unknown sheet names do not get silently canonicalized. Block Do not reconcile. Show unsupported-file reason. 100% certainty we are running the Aetna path.
Schema Gate Missing required columns or drift produces plausible but wrong numbers. parsedWb.errors.length === 0 for required-field failures. Drift is explicit and actionable, not informational only. Block on required columns. Warn on tolerated optional drift. No deterministic-looking output from malformed input.
Redaction Gate Redacted workbook yields universal underpayment noise. Use zero-dollar threshold and stop before audit views if redaction confidence is high. Block reconcile/export; route user to non-redacted upload. No false exception floods from redacted files.

Traceability Gate Exceptions cannot be mapped back to source workbook rows. Every discrepancy must retain sourceSheet/sourceRow references and those rows must exist in raw workbook state. Block source-row export when traceability is incomplete. Each exception is traceable back to raw evidence.

Audit views derive provider/status/drift state from context or display “unavailable”, never fabricated defaults. Block release until demo overlays are removed or clearly labeled as synthetic. Execution Split

10 workstreams for implementation and validation

The execution team should not blend analysis, implementation, and proof into one stream. Split ownership by runtime layer and validation surface so each slice has a crisp success signal.

01

Upload contract

Harden AuditContext.uploadFile around parser errors and supported-file gating.

02

Sheet identity

Stop canonicalizing unknown sheets to Aetna; preserve exact workbook identities.

03

Column mapping

Turn required-column failures into hard blockers with precise diagnostics.

04

Math regression

Add controlled-workbook integration assertions for totals, counts, and discrepancy classes.

05

Generated overlay removal

Remove fabricated provider, drift, clawback, and review-queue defaults from audit views.

06

UI support claims

Align intake copy and watcher behavior with actual Aetna support.

07

Traceable exception export

Export original source rows only for discrepancies using sourceSheet/sourceRow.

08

Annotated XLSX

Add Excel comments for row-level notes; treat border styling as optional until proven.

09

Adversarial gates

Encode fail-fast checks in tests and runtime banners before view rendering/export.

10

Release validation

Run build/tests plus manual workbook proof and exception-export count reconciliation.

Decision: trust is earned at the gate between parse and reconcile, not by polishing dashboards later. The immediate objective is a narrow, exact, adversarially validated Aetna pipeline that proves its math on the controlled workbook and exports only traceable exception rows.
