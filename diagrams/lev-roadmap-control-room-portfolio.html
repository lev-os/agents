North Star Hard Cut

Choose the control-room OS story and make the repo obey it.

The top docs currently carry two products at once: “universal agent framework” and “local-first agent operating system / control room.” The audit says the latter is the real north star. The first portfolio move is to rewrite the canonical story so every runtime, app, and plugin inherits from the same sentence.

What Is RealConcrete Today
  • @lev-os/daemon is a real supervision library with health, worker pools, and dependency management.
  • core/event-bus is real infrastructure with wildcard matching, persistence, and replay.
  • BD -> beads -> publisher is the one actually-concrete write-to-projection path.
  • AgentPing already has dashboard runner and manager-server infrastructure.
What Is Mostly PromisedSpec-First, Not Runtime-True
  • Poly-generated control surfaces under core/poly/src/surfaces/ are described, but not actually present.
  • FlowMind is the substrate in prose, but many shipped flows are not executable end-to-end.
  • apps/voice-api is a heartbeat/skills service, not yet a real voice control plane.
  • The native shell is strongest as spec direction, not as the first MVP delivery path.
Highest Leverage
Rank 1

Poly + Daemon

Convergence, not invention. Make @lev-os/daemon the true lifecycle owner and Poly the true binder/surface owner.

Rank 2

FlowMind + /work

Move from smart skill prose to one canonical execution kernel and one runtime owner for the lifecycle.

Rank 3

Event Contract

Stop paying the schema translation tax across bus, providers, SDLC, and bridge layers.

Rank 4

Storage Authority

Make the “one canonical adapter writes, everything else projects” model real in code, not just in handoffs.

Blunt version: stop creating new product branches until these four substrate cuts exist. You already have enough runtime pieces to dogfood one control-room slice; the missing piece is disciplined sequencing.
