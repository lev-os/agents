04 — What to Steal from OpenSquirrel
01

Delegation Protocol

Fenced-JSON coordinator/worker pattern. The AI decides when to spawn sub-agents, what runtime to use, and what to delegate. App parses and orchestrates.

Port to core/exec adapter
02

Session Persistence

Turn-boundary journaling with pending prompt recovery. Interrupted sessions resume exactly where they left off. Scroll positions, token stats survive restarts.

Port to core/daemon AgentSession
03

Remote Agent Targeting

SSH + tmux session management for distributed agents. Session names and line cursors persist across app restarts for seamless reattach.

Port to lev exec --remote
04

Multi-Runtime Config

TOML-driven runtime definitions with per-agent MCP server wiring. RuntimeDef structs with command, args, env vars, and model flags.

Extend core/config resolver
05 — Decision: Build Into Squirrel

Why building INTO OpenSquirrel is the right call

  • OpenSquirrel is 7,200 LOC and functionally ahead in agent orchestration — subprocess management, GPU rendering, session persistence are done
  • Lev-TUI is 1,700 LOC (terminal) + 1,200 LOC (egui desktop) — clean architecture but no agent execution capability
  • The terminal is not where dev agents live — developers need GPU-rendered, persistent, multi-pane agent sessions
  • OpenSquirrel already handles the hard parts that Lev-TUI hasn't started: subprocess lifecycle, delegation parsing, turn persistence
  • Lev-TUI's modular architecture (6 crates, trait-based) is the decomposition blueprint — OpenSquirrel's monolithic main.rs needs exactly this
  • Keep lev-tui as the ops/CI terminal surface. OpenSquirrel becomes the desktop agent IDE.
The Play: Fork OpenSquirrel → decompose main.rs using lev-tui-core patterns → wire Lev event bus → ship as lev desktop
06 — Risk Register
HIGH
macOS Only

Hard dependency on Metal via GPUI. Linux/Vulkan is an open issue with no implementation path. GPUI's Linux story is immature.

HIGH
Monolithic Structure

7,200 LOC in a single main.rs with god-struct OpenSquirrel. Every feature addition increases cognitive load and merge conflict surface.

MEDIUM
Bus Factor of 1

100% authored by one contributor (Infatoshi). AI-assisted velocity (Cursor) explains speed but raises questions about edge case handling.

HIGH
Security Posture

Every runtime runs with permission bypass flags. No approval queue, no sandboxing, no audit trail beyond transcripts.

MEDIUM
GPUI Stability

GPUI is Zed's internal framework, not a stable public API. Breaking changes could require significant rework with each update.
