Runtime Taxonomy

Surface, Backend, Environment, Modality, and Robot stop being confusing once they occupy different layers.

The convergence is toward an env primitive, but vision and voice are not environments by themselves. They are perception and I/O modalities. The environment is the world the agent can perceive and act in. The backend is the runtime engine. The surface is how humans or external systems talk to Lev.

Recommended layer stack

Read top to bottom. Each layer answers a different architectural question.

surface

How does a human or external system enter Lev?

These are binders and control surfaces. They carry requests in and results out.

CLI MCP HTTP Web UI Voice UI
backend runtime

What engine executes the task?

SDK-backed, CLI-backed, or text-backed runtime. This is where Claude Agent SDK, Pi, OpenCode, and AI SDK sit.

claude-agent-sdk pi-sdk / pi-cli opencode-sdk / opencode-cli ai-sdk-text
environment

What world can the agent perceive and act in?

The environment is the execution world. It bundles available actions, observable state, and lifecycle rules.

env-browser env-gui env-terminal env-toolcall env-robot
modality / perception

What signals does the environment expose?

Vision and voice are not surfaces here. They are capability channels that an environment may expose.

text image audio video telemetry sensor feed
actuation / integration

What can change the world?

Robot arms, drone controls, browser actions, shell commands, and API calls are actuators hanging off the environment contract.

browser click/type shell exec robot motor control drone navigation camera pan/tilt

Recommendation

The cleanest converged model is: surface -> exec/backend -> environment -> modality + actuator.

Vision

  • Not a surface
  • Not an environment by itself
  • A perception capability that an environment can expose

Robot

  • Not a surface
  • Usually an environment family: env-robot
  • Contains modalities and actuators under one contract
Thought exercise result: a robot or drone makes the taxonomy clearer because it obviously has inputs, outputs, world state, and actuation. That same shape scales back down to browser, GUI, terminal, and toolcall environments.

Practical naming rule

  • Surface if it is how a human or external system enters Lev.
  • Backend if it is the runtime engine that executes tasks.
  • Environment if it defines a world with observable state and available actions.
  • Modality if it is a signal channel like image, audio, or sensor stream.
  • Actuator if it changes the environment.
