Risk Analysis
Partial contracts are the highest-risk category. The 4 partial items (HallucinationFilter, ConfidenceScorer, hard-cutover guard, ValidationConfig) represent code that exists and could be assumed to work by anyone reading the specs — but is never actually called in the runtime path. This is worse than missing code: it's a false sense of safety.
  • Rows 11-12: Transcription validation exists (198 tests pass on the data) but the filter and scorer themselves are never invoked by any daemon. A hallucinated transcript sails through unchecked.
  • Row 13: The decision gate emits a violation event when the hard-cutover condition fires, but never actually blocks the response. An unsafe command could still be readback to the pilot.
  • Row 22: The only contradicted element — the spec mandates authority conflict resolution, the config declares the policy, but no runtime code reads it. Active spec contradicts passive implementation.
