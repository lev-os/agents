Leviathan Graph / PromptOS / Nanograph

Prompts Are Converging. The Runtime Story Is Stronger, But The Measurement Story Still Needs Hardening.

This page explains the current state of the Leviathan graph work from three simultaneous angles: PromptOS as deterministic prompt-landscape research, Leviathan as a real but partial graph runtime, and Nanograph as a proven compiler-side export path. The central finding from five refinement passes is that the right next move is not “ship the super-graph,” but “standardize the measurement harness until the graph thesis survives extractor bias.”

1 — What Exists Right Now
PromptOS Research

Deterministic prompt compiler

PromptOS is not a runtime. It is a research compiler that scans prompt artifacts, extracts atoms, normalizes them through a static ontology, and emits YAML, CSV, and GraphML.

  • Local clone under research/system-prompts-and-models-of-ai-tools/
  • Branch: codex/add-promptos at e863c1d
  • Rebuilt locally into out/compiled, out/metrics, and out/graphs
Leviathan Harness Graph

Offline harness compiler

The Python compiler in workshop/extract/leviathan-graph already emits a structural evidence pack over Leviathan docs and runtime modules.

  • Deterministic atom extraction and normalization
  • Compiled YAML plus GraphML similarity/bipartite views
  • Now extended with a Nanograph exporter
Core Graph Runtime

Real, but still V0-shaped

core/graph is not speculative. It owns a live extract → classify → validate → apply path with adapters, bridge code, event storage, classification helpers, and context/projector modules.

  • Canonical LevEvent input
  • Append-only GraphEvent audit shape
  • FlowMind policy seam, but not full policy brain yet

3 — Five Refinement Passes
Pass 1

Baseline read by the six-role team

  • Cloud: the current graph is local-first and in-process, with durability abstracted behind an injected event store.
  • Agentic engineering: core/graph is already a live runtime surface, but FlowMind policy wiring is still more architectural than operational.
  • CEO: the strongest story is “runtime differentiates where prompts converge,” not “we already have a universal graph.”
  • Sales: the wedge is auditable policy + context graphs for agent builders who already feel prompt parity.
  • Marketing: PromptOS is a strong evidence engine; Leviathan is the stronger product thesis, but still too systems-heavy for broad GTM.
  • Branding: substrate, spine, and zoom-lens metaphors help; workflow-engine and prompt-compiler language hurt.
Pass 2

Contradiction and risk hunt

  • Biggest contradiction: “universal context graph” is still larger than the shipped V0 runtime.
  • Biggest GTM risk: density gaps can be read as immaturity, not intentional modularity.
  • Biggest buyer objection: strong architecture story, thin product proof.
  • Biggest messaging risk: selling “context as moat” while context similarity is currently zero.
Pass 3

Pure data-science read

  • PromptOS is high-overlap across multiple slices, not just globally.
  • Harness global density is lower, but the corpora differ in size, ontology, and artifact type.
  • Cross-corpus density is informative, not yet fair.
  • Required next metric: zero-atom rate plus non-zero pairwise Jaccard distributions under matched sampling.
Pass 4

Convergence under constraint

  • Keep the comparative method.
  • Stop treating raw density deltas as thesis-proof.
  • Build a shared evaluation harness over matched corpora.
  • Measure density stability under ablation and bootstrap resampling.
Pass 5

Final one-line recommendations

  • Cloud: standardize extraction and sampling before platform commitments.
  • Agentic: harden the benchmark and wire real policy-driven classification.
  • CEO: validate the thesis with ablation-controlled, native-runtime metrics.
  • Marketing: standardize the extractor and matched-sample comparisons before escalating claims.
4 — Expert Team Readout
Cloud Architect

Local-first, not distributed

Today’s graph is an embedded compositor with an injected store. There is no clear remote graph-service or distributed consistency story in the current slice.

Agentic Engineer

Real runtime, partial policy loop

extract → classify → validate → apply is real. FlowMind as the graph brain is still mostly a seam plus spec commitments.

CEO

Moat is topology, not prompts

The strongest product thesis is not “we have a graph,” but “we have a graph runtime as the state plane of an agent OS.”

Sales

Wedge is auditability

The buyer pain is prompt parity plus context fragmentation. The wedge is auditable policy, provenance, and graph-backed context for agent builders.

Marketing

PromptOS is evidence, Lev is thesis

PromptOS is a compelling content machine. Leviathan is the deeper runtime story, but still too abstract for a broad audience without tighter proof.

Branding

Use substrate, spine, zoom lens

Avoid workflow-engine or prompt-compiler language at the top level. The better frame is a declarative graph runtime for reactive intelligence.

5 — What Is Real vs What Is Still Architectural
Surface Status What Is Real What Is Still Missing
core/graph real runtime Compositor, types, adapters, bridge, event store, classify helpers, context projector, tests. Full FlowMind ClassifyPolicy wiring, richer lifecycle/proposal behavior, stronger view/materialization loop.
PromptOS real research tool Deterministic prompt compiler, ontology, compiled DSL, GraphML and metrics regenerated locally. Not a runtime dependency, not a product moat by itself, and not yet harmonized with harness extraction.
Nanograph POC proven seam Compiler-side export pack emitted and validated through init, load, check, and run. No direct core/graph integration yet. Still an exporter/reference path, not a runtime backend decision.
Universal graph thesis too early to sell Strong docs, handoff lineage, and comparative methodology. Matched ontology, matched sampling, context/policy closure, native dogfood metrics, and stronger product proof.
6 — Recommended Next 30 Days
Decision

Do not widen the claim set yet.

Use the next month to harden the measurement harness, not to declare victory on the universal graph story.

Best single move: one harmonized evaluation runner for PromptOS and Leviathan, with matched ontology, matched sampling, ablation, and bootstrap stability.
Execution Stack
  • Align PromptOS and harness extraction into a shared benchmark harness.
  • Track zero-atom rate and non-zero Jaccard distributions per slice.
  • Add native policy-driven classification events to the current graph runtime.
  • Only then rerun the prompt-vs-runtime differentiation story.
