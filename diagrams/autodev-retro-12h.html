How the Stack Works

Autodev Loop

Recurring prompt (CronCreate) that scans entity surfaces every N minutes. Each tick: SCAN → pick highest-priority item → EXECUTE or DRIFT SCAN → VALIDATE → checkpoint git.

/autodev-loop 10m

lev exec

The execution plane. Runs tasks with adapters and profiles. lev exec --profile sdlc.flowmind.exec composes prompt stacks, runs implementations, tracks tokens.

core/harness/ + core/exec/

lev loop

The scheduling plane. Discovers entities across surfaces, validates fitness functions, promotes completed work to _done/, manages tick budgets.

plugins/core-sdlc/src/commands/loop.ts

SDLC Plugin
