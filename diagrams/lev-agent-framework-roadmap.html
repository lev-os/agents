Strategic Spine

Why this matters

1. Projection, not lock-in

Vaults, docs, and CMS views should be materialized projections. They are how people and agents consume the system, not how the system decides truth.

  • Obsidian becomes a publisher target
  • Graph stays read/project, never upstream write authority
  • Canonical writes flow through adapters and events

2. Grounded research beats generic search

The market is full of “search” and “chat with docs”. The defensible move is durable, cited, local-first synthesis that agents can reuse, inspect, and republish.

  • Source manifests are inspectable
  • Answers carry recoverable evidence
  • Outputs survive beyond one browser tab

3. Agent-native ops need robot mode

If Lev is for agents, the CLI cannot be a pretty shell first and a script surface second. The machine contract needs to be deliberate, dense, and stable.

  • Auto JSON when piped
  • Structured errors with suggestions
  • Meaningful exit codes for orchestration
“The real differentiator is not another agent shell. It is an agent framework where research, memory, projections, and execution all produce durable artifacts with clear authority boundaries.” Framework thesis for Lev

Workflow Loop We Should Own

Repeatable product loop

The article/transcript pattern is valuable because it is operational, not inspirational. It defines a full loop that an agent can execute, audit, and later project into different surfaces.

Step 01

Acquire

Use intake, search, transcripts, and URLs to build the source pool.

Step 02

Curate

Prune to deliberate sources. Signal-to-noise is part of the product.

Step 03

Synthesize

Load into NotebookLM, ask grounded questions, preserve structured refs.

Step 04

Resolve

Extract passages and rewrite citations into local linkable evidence.

Step 05

Project

Publish to Obsidian, CMS surfaces, dashboards, or later graph-derived views.

Why this loop fits Lev better than a vault-specific product story
The CMS handoffs already say the graph is a projection surface and publish targets are adapters. That means the NotebookLM pattern should land in Lev as a research orchestration primitive whose artifacts can be emitted into many targets. Obsidian is simply the first beautiful proof, not the final container.
