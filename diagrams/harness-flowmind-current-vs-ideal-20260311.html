What We Need Next

If you want the highest-leverage ordering from this whole pass, it is this:

1. Stabilize the execution boundary

  • Fix harness module resolution for telemetry.
  • Fix FlowMind executor/session resolution for config XDG imports.
  • Get the critical-path sentinel back to zero unexpected failures.

2. Converge config truth

  • Make `exec` + `flowmind` the only canonical roots.
  • Move docs-only confidence / skill / compaction policy into live config where appropriate.
  • Avoid inventing duplicate roots while the system is still moving.

3. Separate config from product/runtime

  • Repo readiness is a workflow, not a setting.
  • Trust UX is a surface, not a flag.
  • Queue-backed proactivity and judge loops need real runtime services.

4. Use workshop pressure deliberately

  • MASFactory and Droid should sharpen boundary choices, not explode them.
  • Absorb patterns into policy and ontology first.
  • Only after that, build the missing runtime and UX layers.
