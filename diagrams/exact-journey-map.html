Workflow-Driven BDD

User journeys to turn into nuanced behavior-driven tests.

These flows combine the two client vision documents with the routes and service layer that exist today. The goal is not only to prove that screens render, but that the product behaves the way the client imagines work actually happening: wake up, check the dashboard, notice new files, auto-process them, investigate only what matters, resolve it, and report confidently.

BDD Feature Inventory
Feature name Primary persona Core assertion Data contract behind it High-value failure
Feature: Morning dashboard triage Ops Admin / CFO Visible alerts and KPI totals direct the user to the right next workflow. Live reconciliation totals override demo data when available. Dashboard presents stale or mislabeled metrics as current truth.
Feature: Auto-ingest from watch folder Analyst / Ops Admin New workbook is ingested without manual trigger and user is notified of outcome. Watcher emits supported file into upload pipeline once. Silent ingest, duplicate ingest, or no user-visible outcome.
Feature: Schema trust decision Analyst User can decide whether the workbook shape is safe before acting on results. Mapping, drift, raw rows, and parser errors are all exposed. Malformed workbook still produces a believable exception list.
Feature: Exception drilldown Analyst / Reviewer Every visible exception can be traced back to raw evidence and math context. sourceSheet/sourceRow, segment totals, and discrepancy metadata remain linked. User sees an exception but cannot prove where it came from.
Feature: Review queue resolution Review Specialist Status and notes reflect real workflow progress and survive export/handoff. Resolution state is persistent and aligned with visible queue filters. Queue appears operational but state is local-only and vanishes.
Feature: Executive financial readout CFO / Sponsor Topline numbers reconcile to engine totals and support confident reporting. Expected, actual, variance, severity counts, and clawbacks stay mathematically coherent. Executive story diverges from the underlying reconciliation output.
