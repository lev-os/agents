What Changed In The Plan

This is a reclassification, not a deletion
Earlier trajectory

Polymorph-first GenUI

  • Grow primitive set from 12 to 18+.
  • Add schema registry, state, callbacks, streaming, validation.
  • Turn polymorph itself into the main runtime.
  • Build C1-like behavior mostly from scratch.
Corrected trajectory

json-render-first GenUI

  • Adopt upstream generic runtime for catalog/spec/state/actions.
  • Make A2UI an abstract library contract.
  • Translate polymorph / Sofia payloads into that contract.
  • Keep polymorph for compatibility, design artifacts, or later non-web paths.
Doing `json-render` first does not supersede polymorph. It changes polymorph from “future primary runtime” into “existing asset with a narrower, cleaner job.” practical interpretation

Roadmap To GenUI

Ordered to stabilize the abstract contract first

This is the path that honors the correction. Build the abstract library boundary first, then the canonical host, then real surfaces, then streaming/state depth, then legacy convergence.

Phase 0

Freeze the ownership model

  • Lev owns shell, layout, policy.
  • AgentPing owns the GenUI host bridge.
  • A2UI is the abstract library contract.
  • json-render is the first concrete implementation under that contract.
Phase 1

Define one A2UIEnvelope

  • One abstract input contract for all rendered interfaces.
  • Fields should cover surface, spec, state, actions, meta.
  • CanvasRenderer becomes a translator into this envelope, not the final renderer.
Phase 2

Promote the canonical resolution index

  • Grow src/catalog.tsx into the primary component registry.
  • Keep composition spec-first: primitives plus composite builders.
  • Stop adding new manual resolution logic elsewhere.
Phase 3

Ship four real surfaces

  • Approval sheet
  • Diff viewer
  • Metrics board
  • Config form
  • These are already encoded in task lev-dfbv.
Phase 4

Add runtime depth

  • Streaming patch transport
  • Session state and action loops
  • MCP-hosted inline surfaces
  • dashboard-runner hosted GenUI views
Phase 5

Converge legacy inputs

  • Translate polymorph templates to the envelope where useful.
  • Keep Pencil / design-export paths if they still earn their keep.
  • Delete duplicate resolution paths only after the host path is proven.
