03

Highest-Value Enrichment Moves

The right move is selective enrichment by composition. These are the smallest additions that would make the current setup feel as smooth as the upstream toolkit without sacrificing Lev’s stronger architecture and runtime model.

Move 1
Reusable project worktree contract

Package deterministic setup, environment copy, dev port allocation, and fail-fast verification for normal app repos. This is the cleanest missing operator surface.

Move 2
Checkpoint review workflow

Compose local verification, static analysis, browser/auth checks, optional second-opinion review, and defer/escalate behavior into one deterministic workflow.

Move 3
Docs drift sync

Add a thin post-commit workflow that inspects the diff, identifies stale README or CHANGELOG expectations, and suggests or applies bounded documentation updates.

Move 4
Compact task verifier

Create a small-radius acceptance-criteria verifier for changed files and expected tests, without requiring the full PM artifact lifecycle every time.

04

Guardrails

Do not import the upstream mental model wholesale.
The upstream toolkit is optimized around project-local .claude skills and canonical markdown artifacts like PRODUCT_SPEC.md, TECHNICAL_SPEC.md, and EXECUTION_PLAN.md. Your current stack already has stronger canons through .lev/pm/*, FlowMind contracts, and bead-first tracking. Borrow the packaging, not the ontology.
