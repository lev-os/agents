## Scoring Rubric (0-50 Points)

### Five Criteria (10 points each)

**1. Practitioner Weight**
- 10: Built/shipped products with this framework
- 5: Teaches from case studies
- 0: Pure theory, no field experience

**2. Clarity & Executability**
- 10: Step-by-step, immediately actionable
- 5: Requires interpretation
- 0: Philosophical, not executable

**3. Proven ROI**
- 10: Multiple documented success stories with metrics
- 5: Anecdotal evidence
- 0: Unproven

**4. Novelty & Non-Obviousness**
- 10: Counter-intuitive hidden gem
- 5: Useful synthesis
- 0: Obvious or trivial

**5. Cross-Domain Applicability**
- 10: Universal pattern across all domains
- 5: Works in 2 domains
- 0: Highly specialized

### Score Ranges
- **45-50**: Canonical, must-include
- **35-44**: High-value, strong candidate
- **25-34**: Valuable, include if space permits
- **15-24**: Niche, consider for completeness
- **0-14**: Low priority, exclude

## Framework Inclusion Checklist

### Must Pass All Five:
1. **Clarity**: Can explain in 2 sentences?
2. **Execution**: Can write 3-7 actionable steps?
3. **Differentiation**: Distinct from other frameworks?
4. **Value**: Would experienced practitioner find useful?
5. **Source**: Credible practitioner or proven?

### Red Flags (Exclude):
- Vague steps ("think about X")
- Obvious checklists
- No clear usage trigger
- Purely academic, no implementation
- Rebranded existing concept
- Consultantware (designed to sell consulting)
- Buzzword-heavy, no substance

2. **Executable > Philosophical**: Clear steps > abstract principles
3. **Proven > Novel**: Track record > untested ideas
4. **Sparse > Comprehensive**: Essential steps only, expand later
5. **Cross-domain > Specialized**: Universal patterns valued higher
6. **Hidden Gems > Common Knowledge**: Non-obvious insights prioritized
