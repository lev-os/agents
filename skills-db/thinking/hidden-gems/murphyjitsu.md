# Murphyjitsu - CFAR's Adversarial Planning Technique

**Source**: CFAR (Center for Applied Rationality) / LessWrong
**Year**: 2010s rationality movement
**Obscurity**: Tier 2 (⭐⭐) - High (rationality community)
**Domain**: 06-problem-solving / 13-management-strategy
**Score**: 40/50

## Origin & Context

Murphyjitsu is a planning technique developed by the Center for Applied Rationality (CFAR) designed to strengthen plans by repeatedly envisioning and defending against failure modes until you would be shocked to see it fail.

The technique emerged from addressing what CFAR designers saw as pervasive "disembodiment" in the rationality community - people knew theories but didn't act on them. Murphyjitsu blends pragmatic pessimism with drive toward action, functioning like a martial art where you see future problems coming and actively counterstrike them.

## Core Principle

**Assume project fails, simulate Murphy's Law preemptively**

Action-oriented pessimism - not just predicting failure, but actively preparing to fight against it.

## Core Principles

1. **Prospective Hindsight**: Imagine failure already occurred and work backward (increases ability to identify causes)

2. **Action-Oriented**: Not just prediction - develop specific preventions for each failure mode

3. **Iterative Adversarial Simulation**: Repeatedly generate failure modes (~17 in 5 minutes)

4. **Implicit Knowledge Access**: Taps gut-level knowledge about how plans typically fail

5. **Murphy's Law Mindset**: "Anything that can go wrong, will go wrong" - prepare accordingly

## The 6-Step Murphyjitsu Process

### Step 1: Pick a Plan/Project/Goal
**What**: Select something concrete that might fail
**How**: Choose specific objective with clear next action
**Output**: Defined plan to test
**Example**: "Write blog post by Friday," "Raise Series A," "Ship game project," "Go to gym tomorrow"

### Step 2: Make Specific Enough to Visualize
**What**: Identify next concrete action to keep project moving
**How**: Not vague aspirations but actual physical actions you can picture
**Output**: Visualizable next step
**Example**: Not "work out more" but "wake up at 6am, put on gym clothes, drive to gym"

### Step 3: Check Your Surprise-O-Meter
**What**: Visualize putting plan in motion - how surprised if it failed?
**How**: Imagine plan execution and ask "Would I be shocked if this didn't happen?"
**Output**: Surprise level assessment
**Example**: If genuinely shocked by failure → done; if not surprised → continue to Step 4

### Step 4: Use Pre-Hindsight
**What**: The plan didn't work - it failed specifically at the next action
**How**: Imagine future self appears: "No! Don't do it! You'll fail!" then disappears. What's most probable failure mode?
**Output**: Specific failure scenario
**Example**: "Stayed out late with friends and forgot" or "Woke up but felt too tired" or "Got to gym but it was closed"

### Step 5: Use Looking Forward
**What**: Having identified failure mode, what action prevents this specific problem?
**How**: Visualize taking preemptive action - does it defuse danger or create new weak point?
**Output**: Concrete prevention measure
**Example**: Set specific time to leave friends (10pm cutoff) or set alarm across room or verify gym hours

### Step 6: Iterate
**What**: Return to Step 3 with updated plan including prevention
**How**: Check surprise-o-meter again - still not shocked? Generate next failure mode
**Output**: Cycle until genuinely surprised failure could occur
**Example**: Generate ~17 failure modes in 5 minutes, each with simple preparation/contingency

## Murphyjitsu vs Pre-Mortem: The Key Distinction

While both use prospective hindsight (imagine failure, look backward), they serve different purposes:

### Pre-Mortem
- **Primary Function**: Predictive tool for deciding WHETHER to proceed
- **Goal**: Calibrated probability estimates
- **Emphasis**: Realistic risk assessment
- **Use Case**: Go/no-go decisions

### Murphyjitsu
- **Primary Function**: Action-oriented tool for ensuring success
- **Goal**: Identify and eliminate failure modes
- **Emphasis**: Martial/adversarial preparation
- **Use Case**: When already committed, need to succeed

**Key Quote**: "Reference class forecasting may provide better calibrated probability estimates, but Murphyjitsu prioritizes moving your 'hands and feet'—getting things done despite acknowledged risks."

## How It Surfaces Hidden Failure Modes

### Mechanism

**Tap Implicit Knowledge**: Your brain has absorbed countless patterns about how plans derail through lived experience. Murphyjitsu provides structured framework for converting tacit knowledge into actionable insights.

**Systematic Adversarial Imagination**: By asking "What would have to happen for this to fail at this specific step?" you engage intuitive knowledge system that conscious planning might miss.

**Iterative Revelation**: After addressing one failure mode, you've created new weak points. Cycling through Steps 3-6 repeatedly progressively strengthens plan against increasingly subtle scenarios.

**Gut-Level Pattern Recognition**: The "surprise-o-meter" calibration accesses your implicit understanding of what typically goes wrong, not just what you consciously expect.

## When to Use Murphyjitsu vs Alternatives

### Use Murphyjitsu When:
- Implementation-focused planning (need to ensure execution)
- Already committed to action, want to maximize success
- Personal/team projects where you control variables
- Quick turnaround needed (5-15 minutes)
- Asking "How do I make sure this succeeds?"

### Use Pre-Mortem When:
- Need calibrated probability estimates
- Deciding WHETHER to pursue something
- High-stakes go/no-go decisions
- Team alignment on risks before commitment

### Use Reference Class Forecasting When:
- Need better time estimates
- Historical data exists for similar projects
- Outside view matters more than inside view

### Use Red Teaming When:
- Have resources for dedicated adversarial team
- High-stakes scenarios justifying extensive analysis
- Organizational context (not personal planning)

## Integration with Rationalist Techniques

Murphyjitsu operates alongside other rationality frameworks:

**Reference Class Forecasting**: System 2 (deliberate) using explicit historical data
**Back-planning**: Start from desired outcome, work backward to identify necessary steps
**Inside View**: Your intuitive default planning (tends toward optimism)
**Murphyjitsu**: Adds structured pessimism to intuitive process

**Combined Approach**:
1. Reference class forecasting → realistic time estimates
2. Back-planning → comprehensive step mapping
3. Murphyjitsu → identify and defend against failure modes in implementation

This addresses planning fallacy from multiple angles.
