# ARIZ (Algorithm for Inventive Problem Solving)

**Source**: Genrich Altshuller - Soviet engineering
**Year**: 1956-1985 (30+ years development)
**Obscurity**: Tier 1 (⭐⭐⭐) - Very High (advanced TRIZ, Soviet)
**Domain**: 06-problem-solving
**Score**: 42/50

## Origin & Context

ARIZ represents Altshuller's most advanced TRIZ tool, designed for ultra-complex technical problems where the 40 principles and contradiction matrix fail. Developed across 30 years, it's an approximately 85-step algorithmic process for systematically deconstructing and reformulating complex problems until they become solvable.

Only about 5% of engineering problems are complex enough to warrant ARIZ - the other 95% should use basic TRIZ tools (40 principles, contradiction matrix).

## Core Principles

1. **Solution-Neutral Process**: Deliberately strips away preconceived solutions from problem statements ("fresh pair of eyes")

2. **Problem Reformulation Focus**: More than 50% of ARIZ is dedicated to reformulating the problem itself

3. **Contradiction Escalation**: Works systematically from technical contradictions to physical contradictions

4. **Integrated Tool Synthesis**: Combines ideality, contradictions, resources, scientific effects, S-field modeling, standard solutions, and 40 principles

5. **Algorithmic Rigor**: Step-by-step logical sequence of questioning and reinterpretation

## How ARIZ Differs from Basic TRIZ

### Philosophical Approach

**Basic TRIZ (40 Principles)**:
- Pattern matching to historical inventions
- Solution-focused (here are proven answers)
- Taxonomic classification (map problem → contradiction matrix → solutions)

**ARIZ**:
- Solution-neutral algorithmic process
- Problem-focused (understand the problem deeply first)
- Continuous reformulation until solvability emerges

### Complexity and Application

|--------|-----------|------|
| **Learning Curve** | Accessible with training | Requires mastery |
| **Application Rate** | 95% of daily problems | 5% of unique problems |
| **Time Investment** | Minutes to hours | Hours to days |

### When Each Applies

**Use Basic TRIZ when**:
- Problem can be clearly categorized as technical trade-off
- Historical analogs likely exist
- Speed matters (quick solutions needed)
- Standard contradiction matrix provides viable principles

**Use ARIZ when**:
- 40 principles yield no viable solution
- Problem is genuinely unique (no historical analog)
- Multiple interconnected contradictions resist decomposition
- Novel technology terrain where patent analysis insufficient
- High-value problem justifies significant time investment

## The ~85-Step Algorithm Structure

While the complete 85 steps aren't enumerated in available sources, the core procedural framework includes:

### Phase 1: Problem Definition and Reformulation
- Clear problem statement
- Identify inherent contradictions
- Strip away preconceived solution assumptions
- Reformulate problem multiple times to reveal true nature

### Phase 2: Contradiction Identification and Evolution
- Distinguish technical contradictions (parameter trade-offs)
- Identify physical contradictions (opposing requirements)
- Systematically escalate from technical to physical
- Reformulate at each contradiction level

### Phase 3: Resource and Scientific Effect Integration
- Identify system and environmental resources
- Match resources to applicable scientific effects
- Constrain solution space to actually available resources

### Phase 4: Solution Generation via S-field Modeling
- Apply Substance-Field modeling
- Use standard solutions
- Integrate 40 principles when applicable
- Generate workable configurations

### Phase 5: Solution Refinement
- Test solution concepts
- Verify contradiction elimination (not merely shifted)
- Check for new contradictions introduced
- Align with technical system evolution laws

## Step-by-Step Application

### Step 1: Define Problem Without Solutions (15 min)
**What**: State technical challenge, strip away assumed solutions
**How**: Describe what must be achieved, not how to achieve it
**Output**: Solution-neutral problem statement
**Example**: Not "make material stronger without increasing weight" but "material must simultaneously be strong and light"

### Step 2: Identify All Contradictions (20 min)
**What**: Find technical (parameter trade-offs) and physical (opposing requirements) contradictions
**How**: Map what improves vs what worsens; identify conflicting requirements
**Output**: Complete contradiction inventory
**Example**: Strength increases → weight increases (technical); Material must be both rigid and flexible (physical)

### Step 3: Reformulate Problem (30 min)
**What**: Reinterpret problem through multiple lenses until true nature revealed
**How**: Ask "why" repeatedly, examine assumptions, consider alternative framings
**Output**: Deeper problem understanding
**Example**: "Low fuel efficiency" reformulates to "energy losses in combustion" reformulates to "incomplete fuel oxidation"

### Step 4: Map Resources (20 min)
**What**: Identify all available system and environmental resources
**How**: List materials, energy sources, fields, spaces, time periods available
**Output**: Resource inventory constraining solution space
**Example**: Waste heat, gravitational fields, pressure differentials, chemical by-products

### Step 5: Apply Ideality Concept (15 min)
**What**: Envision Ideal Final Result unconstrained by current limitations
**How**: Define maximum benefit with minimum cost/harm
**Output**: Aspirational target guiding solution search
**Example**: "Device performs function automatically using free environmental resources with zero maintenance"

### Step 6: Match Resources to Scientific Effects (30-60 min)
**What**: Find scientific principles enabling resource utilization
**How**: Consult scientific effects database, apply physics/chemistry/biology knowledge
**Output**: Potential solution mechanisms
**Example**: Thermal expansion for actuation, piezoelectric effect for sensing, catalysis for reactions

### Step 7: Generate Solutions via S-field and Standards (30 min)
**What**: Apply Substance-Field analysis and standard solutions
**How**: Model system as substances + fields, apply proven transformation patterns
**Output**: Candidate solution concepts
**Example**: Add mediating substance, change field type, utilize existing field

### Step 8: Integrate 40 Principles (20 min)
**What**: Apply relevant inventive principles to solution concepts
**How**: Consider which of 40 principles could enhance solution
**Output**: Refined solution approaches
**Example**: Segmentation, nested doll, mediator, parameter changes

### Step 9: Verify and Refine (30 min)
**What**: Test if solution genuinely eliminates contradictions
**How**: Verify no trade-offs remain, check for new contradictions
**Output**: Validated solution or return to earlier steps
**Example**: Confirm material actually achieves both strength and light weight without compromise
