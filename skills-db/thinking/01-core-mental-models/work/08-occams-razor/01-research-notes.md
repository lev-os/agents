# Research Notes: Occam's Razor

## Origin
- **Author**: William of Ockham
- **Year**: 14th century (1287-1347)
- **Source**: Philosophical writings, though exact quote attribution varies

## Core Concept
Occam's Razor (Principle of Parsimony): When choosing between competing explanations that make the same predictions, prefer the simpler one requiring fewer assumptions. The original Latin: "Pluralitas non est ponenda sine necessitate" (Plurality should not be posited without necessity). This is NOT about choosing simple over complex, but about avoiding unnecessary complexity when simpler explanations work equally well.

Often misquoted as "Entities must not be multiplied beyond necessity" (Ockham never wrote this exact phrase).

## Key Insights for SKILL.md
- ONLY applies when competing explanations have equal predictive power
- It's a heuristic (rule of thumb), not proof
- Prevents adding unnecessary assumptions (entities, complexity)
- Common applications: debugging (check simple causes first), medical diagnosis (common diseases first), scientific theories
- Anti-pattern: oversimplifying when complexity is actually needed
- Modern applications: system design (KISS principle), API design, architecture decisions
