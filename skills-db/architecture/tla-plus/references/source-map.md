## What each source is best for

- Lamport pages: conceptual framing, official learning routes, and tool roles
- Wiki: practical usage, TLC CLI, VS Code guidance, and community entry points
- `tlaplus/tlaplus`: current CLI surface and maintained tool behavior
- `tlaplus/Examples`: validated specs, model files, and reusable case studies
- `vscode-tlaplus`: workflow guidance and knowledge-base articles on common mistakes
- TLAPS docs and repo: proof-manager scope and proof-system workflow
- Apalache docs and repo: symbolic checking assumptions and tutorial path
