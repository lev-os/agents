THE TRIO

/dump · /capture · /sitrep

Three skills, one substrate. The user's daily drivers.

🧠

/dump

Deep mode of /capture. Brain dump on everything — inventory + auto /research + /prior-art per topic + dispatch shovel-ready to /exec + /propose the rest.

Nothing stays in memory.

Calls capture.sweep(mode=deep) — runs the fidelity convergence loop with prior-art enrichment. Max 3 iterations without fidelity gain → halt + note gap.

Output: durable YAML in .lev/pm/workstreams/<ws>/captures/YYYYMMDD-*.yaml

📥

/capture

Standard mode inventory sweep. Classify → route → fidelity score → write.

Order: SAVE first, THEN show the delta.

Fidelity formula: 0.30·detail + 0.25·rel + 0.20·attr + 0.15·nbr + 0.10·action. Gate ≥ 0.8 to proceed to /propose.

Emits trigger: audit://capture/<ws>/<ts> (or collapse://ws/<id> under the emote model).

Calls capture.sweep(mode=standalone).

📊

/sitrep

Freestyle dashboard. Reads the trace stream. Never writes.

Emits: projection://sitrep/<ws>/<ts> — a view primitive (read-only per dna/graph.yaml).

Projects 5 sections: exec brief · active entities · memory state · next actions · decisions needed. Inline emote dispatch (expand 2, collapse 4,5,6, admit 3).

Theater check: flags any /close trigger with items != [] as ⚠.

Source: .lev/pm/workstreams//trace/.yaml (NOT handoff archaeology).

MINORITY REPORTS

Dissenting signals preserved

These didn't win, but kill them and the design breaks

Trust Q1 most (0.78)

Q1 broke the 0.82 cluster with the strongest code evidence — direct citations to compositor.ts:237-333 and dna/graph.yaml:12-33.

When 4 artifacts cluster at 0.82 and one defects with code citations, the defector is the signal, not the noise.

T2 synthesis flagged this: "Q1 at 0.78 breaks the pattern and is the most defensible."

Emotes fill epistemic slot

Q3 showed admission://, question://, hypothesize:// are not just emotes-not-verbs — they fill an epistemic-state slot verbs structurally lack.

The lifecycle state machine has no verb for "I was wrong" or "I'm guessing."

Do NOT collapse emotes into verbs in the schema — keep schema.kind: emote as distinct from schema.kind: verb in lifecycle_trigger.v1, even after behavioral unification.

Q2 could be 0.6 not 0.82

HYBRID adds capture.sweep() infrastructure for a verb the user signaled will fold into emotes.

Possible waste — T3-B's PR 4 may demolish what PR 2/3 built.

Recommend: ship PR 1 only (shared include + drift patch, zero behavior change); defer PR 2-4 until emote-collapse decision.

Q5 masks political risk

"Ship overlays first, non-negotiable" is technically correct but assumes the team resists beads' appeal — judgment, not architecture.

T3-C co-exist pattern defangs the politics: we are not picking. Overlays ship as schema; beads is deferred projection layer, not competing backend.

Nobody loses. The tension was illusory.

THE UNDERLYING INSIGHT

We are applying existing DNA, not inventing new architecture.

Filesystem = substrate, beads = projection — mirrors dna/graph.yaml's own entities-vs-markdown-projections ADR. Overlays ship before beads because contract must precede backend. Emotes are verbs seen through a different axis — same Acts, different directions. Capture and close share a subroutine because they ARE the same operation at different ceremony levels. Every convergence in this session resolved by finding the existing DNA pattern the design was already trying to become.
