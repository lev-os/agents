Onboarding, Install, and Project-Init Matrix

Evidence-backed comparison across Lev, Hermes Agent, OpenClaw, Oh My OpenAgent, Oh My Codex, and Oh My ClaudeCode using workshop intake docs plus two local lev.exec scouts.

sources: workshop/intake/*, workshop/analysis/*, docs/specs/spec-onboarding.md, /tmp/lev-scout-cluster-a.md, /tmp/lev-scout-cluster-b.md

TAKEAWAYS

What Each Platform Optimizes For

The best onboarding experience depends on whether the system is a gateway, a plugin, or a workflow layer.

🦞

OpenClaw

Most complete operator onboarding. openclaw onboard plus a very deep doctor/migration surface, daemon install, control-plane runbook, and explicit security defaults.

☤ Hermes

Hybrid script + wizard + gateway model. Strong migration story from OpenClaw, profile-aware homes, and broad provider portability, but less consumer-polished than OpenClaw.

🧠

OpenAgent

Most explicit provider questionnaire. Install guide asks subscription-by-subscription questions, drives auth setup, and writes agent-specific model config rather than one global provider default.

⚙️

OMX

Best scoped setup. omx setup clearly owns prompts, skills, config, AGENTS.md, .omx/ runtime state, and user vs project scope. Onboarding cleanly hands off into the canonical workflow.

🧩 OMC
Fastest plugin-first quick start, but the install story is split: marketplace/plugin path, npm path, local setup, and variant repos all coexist. Good UX, weaker source-of-truth discipline.

🜂 Lev

Cleanest conceptual split: lev install for machine infra, lev init for project bootstrap, install() SDK for lifecycle orchestration. Least productized operator surface today.
