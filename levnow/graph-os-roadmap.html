ARCHITECTURE

Poly Meta-Substrate → graph-os

The 5-layer pattern maps perfectly

Lev's strongest subsystems converge on: **declaration → compiler → IR → runtime → federation**. This is not metaphorical — each layer has a concrete implementation:

**Declaration** = config.yaml, .flow.yaml, dna/*.yaml (the authored truth) **Compiler** = lev install / config resolver (normalize, validate, lower) **IR** = entity graph + event store (stable form, 944 entities, multiple consumers) **Runtime** = graph-os.ai / local Lev / ClawBuddy / kingly.cloud (projections) **Federation** = graph.store mesh (5th substrate layer — discovery, composition, credit settlement)

PROTOCOLS

graph-os Must Speak

MCP (Anthropic)

Unanimous adoption. Every framework, every tool, every agent runtime supports MCP. Lev already has MCP adapters. Non-negotiable.

A2A (Google, 150+ orgs)

Agent-to-agent interop. Linux Foundation governance. graph.store nodes should be A2A-addressable. This is how federation talks to the outside world.

AG-UI (Microsoft)

Emerging. Watch but don't commit. Agent-to-UI protocol. May matter for ClawBuddy's assistant surface. Low urgency.

FEDERATED MESH

The Non-Crypto Gap

Crypto proves the model. graph-os removes the blockchain overhead.

Hyperspace has 2M+ nodes. Bittensor has $712M in subnets. Akash has $4.3M ARR. They all prove federated mesh works. But they all require crypto tokens. graph-os fills the gap: credit-based incentives, no blockchain overhead, same economic model.

EXECUTION

4-Phase Roadmap

Ship B (unified ClawBuddy), grow into C (graph-os platform)

Phase 0: Foundation (rename + core features). Phase 1: Product (assistant + AgentPing + kingly.cloud standalone). Phase 2: Platform (graph-os.ai portal + cloud mode + A2A). Phase 3: Federation (mesh + credits + enterprise).

TIMELINE

Foundation

Federation

graph.store federation protocol. Mesh inference. Credit incentives. Enterprise private nodes. graph-os.ai managed dashboard.

Phase 4 (Mo 6-12)

Scale

Agent Trials (E2B sandbox). Bundles. lev-forge CI/CD. White-label graph.store. iOS App Store launch.

BOTTOM LINE

One Binary, One Config, Every Mode

Lev is everything. graph-os.ai is Lev with a cloud config.yaml. kingly.cloud is the pipes. graph.store is the marketplace. ClawBuddy is the consumer face. Everything works locally. Cloud is optional. This is the selling point. The only agent runtime where one YAML declaration generates every surface and executes under formal constraints — not a workflow builder, a physics engine for agents.
