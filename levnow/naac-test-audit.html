VERDICT — TL;DR

Three findings, one fix list

FINDING 1 — Five files lie about production

Top 5 mock-heavy unit tests carry ~80% of all mock load. The worst patches modules.intent.daemon.create_broker AND DispatchEngine.from_config — so a real wiring regression doesn't flip the test red. These are the 'faster tests that lie'.

FINDING 2 — 'demo' means five different things

test-demo (integration test), demo-dump (scoring tool), baseline-demo (scorecard wrapper), gate-demo-mode-prod (profile guardrail), voice-demo-up (runtime stack). Same anti-pattern that produced the baseline-* sprawl already pinned in CLAUDE.md.

FINDING 3 — Docker test lane is vestigial

test-nats-e2e / test-audio-e2e / test-golden all spin Docker NATS. FakeDigiRig adapter (tests/integration/helpers/) already gives you the right surface. Docker lane predates that adoption. Collapse it.
