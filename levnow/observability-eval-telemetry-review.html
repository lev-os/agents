ARCHITECTURE

Proof Chain

Intent

MonitorCheck or HeartbeatObservation enters with owner, target, schedule, and freshness.

Run

A bounded probe emits RunEvidence tied to run identity.

Trace

Trace exposes monitor id, verifier, exit code, and evidence refs.

GateProof

Semantic acceptance fails closed for stale, fake, or missing evidence.

Projection

Status renders from proof-backed state only.

PROPOSAL

monitor-heartbeat-proof-spike-v1

The POC slices are locally verified; production promotion is intentionally paused for review.

S1 Contract POC

Monitor and heartbeat contracts reject missing target, stale timestamp, fake success text, missing evidence refs, and bounded live probe timeouts.

S2 Run Evidence

RunEvidence, trace, structural receipt, telemetry capture identity, and semantic GateProof form one queryable proof chain.

S3 Status Projection

Projection shows healthy only from fresh proof and degrades or blocks stale and missing proof.

S4 Review Gate

Pentagon, UltraQA, test triad, package typechecks, receipt parsing, and worktree isolation now pass locally; submodule promotion remains before merge.
