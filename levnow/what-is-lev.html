Agents are trapped in chat windows

The interface hasn't changed since IRC.

Every team rebuilds the same scaffolding, badly, and locks it inside one vendor's cloud.

Lev exists to end that. Not by building a better chatbot — by building the runtime the chatbot should have been running on all along.

THE SHAPE OF THE RUNTIME

Three planes, one spine

Everything meaningful in Lev is graph-shaped — context, execution, skill composition, knowledge, and provenance.

🧬

FlowMind — Control & Policy

A declarative YAML substrate that compiles intent into executable graph documents. Constraints, policies, agent behaviors, entity lifecycles, even memory decay are declarations, not code. FlowMind is the CSS for agents — and the kernel governs itself through the very same engine it gives you.

🕸️

Graph — State & Knowledge

Context is a graph. Execution is a graph. Provenance is a graph. But Lev rejects one-storage-rules-them-all: code facts use AST indexes, operational state uses tables and queues, events carry replay and proof. The invariant is gate-checked mutation with receipts — not a universal database.

⚡

Event Bus — Causality

Every cross-plane boundary emits a canonical LevEvent with correlation IDs — 30+ typed event types across exec, daemon, supervisor, worker, and task domains. Auditable by construction. You can replay the whole system's reasoning, not just read its logs.

🎛️

Orchestration — Execution

Hard boundaries, no god-module. FlowMind never dispatches workers. Orchestration never mutates policy. Graph never schedules. This separation is exactly what kills every other agent framework at scale — and exactly what Lev refuses to violate.

TECHNICAL BRIEF

The architecture, for the people who'll read the source

Seven pillars. Each one is a design decision other frameworks bolted on as an afterthought — and Lev made native.

FlowMind

What: a YAML control plane that compiles to executable graph documents.

Why it's exciting: constraints are declarations, not if statements. The kernel evaluates its own policies through the same engine that runs your flows. "It's the programmable substrate for a self-governing epistemic operating system."

DNA Contracts

What: behavioral contracts (.dna.yaml) that compile to executable gates.

Why it's exciting: "Code is an artifact of DNA, not the other way around." Intent .md → contract .dna.yaml → execution .flow.yaml. Tests are generated from constraints. Proven empirically: 14/15 treatment vs 0/15 control across Claude + Gemini.

Fractal Filesystem

What: the filesystem is the discovery mechanism.

Why it's exciting: no central registry, no DI container. Drop a config.yaml, the scanner finds it. Deterministic cascade merge: system < global < routing < project < project-local < profile < env. Same model for one module and a federated mesh.

Poly — Protocol + Runtime

What: a types-only inter-runtime contract (PolyCard, PolyOp, PolyFrame, PolyReceipt) plus a runtime that compiles + dispatches.

Why it's exciting: one SDK compiles to CLI, MCP, HTTP, gRPC, and WebSocket surfaces from a single registry. Zero hand-written adapters.

The Constraint Manifold

What: five non-negotiable kernel axioms — Finitude, Non-Commutation, Nominalized Reality, Ratchet, Locality — loaded at boot as immutable System FlowMinds.

Why it's exciting: these aren't configuration. They're the kernel's own programs, evaluated through its own substrate. Fail-closed: cannot evaluate → DENY.

Workstreams & Graph Patches

What: Conversation → Understanding → Alignment → Graph Patch → Execution.

Why it's exciting: a patch is multidimensional — many parts of the graph change at once, so multiple workstreams run without serialization. Ephemeral ideas get promoted to durable entities: lev://entity/{space}/{type}/{id}.

The Meta-Harness

What: Lev defines and runs its own constraint harnesses.

Why it's exciting: "90% of cases: Lev IS the harness. 10%: custom. But you define the DNA first, and the cascade builds code around the contract. Prompts converge. Harnesses differentiate."

TRUST IS ENGINEERED, NOT PROMISED

Constraints that never bend

Fail-closed, not fail-open. If the kernel can't evaluate a policy, the action is denied. The ratchet only tightens.

🎫

Leases, not permissions

Everything expires. Trust is earned through demonstrated competence — never hardcoded. An agent proves itself before its scope widens.

🔒

Fail-closed kernel

Cannot evaluate a policy = DENY. No inference, no repair, no silent failures. The safe default is the only default.

⚙️

The Ratchet

State transitions are irreversible. Constraints only tighten, never relax — and tightening requires evidence through admission gates. The audit trail is append-only.

👑

Sovereignty by default

XDG-compliant paths, scoped AgentGuard permissions, PII containment. Local-first, self-hostable, no vendor lock-in, no API keys required.

🚪

Admission boundary

The hard cut where probabilistic belief meets deterministic execution. Inference and trust scoring happen at admission only — never inside a compiled execution graph.

✂️

The translator boundary

Probabilistic classification above the line, deterministic evaluation below. The kernel never sees confidence scores or raw natural language. No inference crosses.
