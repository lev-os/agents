Lev SDK, FlowMind, Poly Surfaces, and Mesh

The product target is an operated adaptive agent runtime: a .now GenUI page can launch an interview, ACP/WS back into Lev, execute through SDK → FlowMind → PolyCard → tmux or endpoint provider, and stream receipts/evals back to the UI.

Architecture

The active spine

graph LR
  SDK[SDK intent] --> FM[FlowMind policy/control]
  FM --> POLY[PolyCard / PolyOp RegistryRuntime]
  POLY --> BD[BackendDescriptor]
  BD --> TMUX[tmux AgentSession]
  TMUX --> EVT[LevEvent / PolyFrame stream]
  EVT --> REC[PolyReceipt + graph memory]
  REC --> MESH[Future mesh node reputation]

Do not create a second runtime. The SDK projects into the same execution chain as CLI and MCP.

Known cutover bug

core/sdk/src/index.ts currently mirrors substrate into host, and core/sdk/src/contract-matrix.ts maps substrate_policy to CLI host. The next implementation slice must hard-cut tmux:// to substrate-only.

MVP

The minimum convincing loop

1. Launch

SDK/CLI/MCP starts an arch interview or FlowMind run with substrate: 'tmux://...'.

2. Resolve

FlowMind profile and PolyCard registry resolve a BackendDescriptor before runtime.

3. Operate

tmux AgentSession owns live CLI lifecycle. Humans can attach and kick the tires.

4. Stream

SDK receives text/tool/usage/context-pressure/checkpoint events as LevEvent/PolyFrame projections.

5. Control

Session handle supports wait, cancel, attach, capture, and typed receipts.

6. Remember

Graph records receipts, provenance, relationships, and future mesh reputation signals.

Boundary

sdk.agent.session() should feel like Claude/Cursor/Flue, but it must be a facade over @lev-os/exec, not a second runtime owner.

Adaptive runtime

Workflow includes declared DAGs, one-shot runs, and bounded agent autonomy

Declared workflow

FlowMind graph/YAML or sdk.flow.define(); topology is known before execution; gates enforce node transitions.

One-shot run

sdk.agent.run() or one-node FlowMind; provider/workspace/tool/result gates still apply.

Adaptive session

Agent explores within constraints; Lev enforces at leases, tool calls, filesystem writes, tests, budgets, stop conditions, and receipts.

Design caveat

FlowMind is not only static workflow DSL. It is also the policy/gate envelope around adaptive agent runtime behavior.

Fail-closed rule

An eval run without receipts, explicit gates, or deterministic proof is not complete. Constraint suites are promotion gates, not dashboard decoration.

NAAC

Anti-cheat and final-boss prior art

Deterministic probes

naac-anticheat scans for custom normalization, mock boundary cheating, reverse imports, raw scenario loaders, and FMM fail-closed regressions.

Adversarial model review

Agentic scan checks canned outputs, verdict smoothing, conditional overrides, and other cheats grep cannot catch.

Final boss gate

naac-final-boss runs preflight, deterministic system validation, adversarial review, then accepts or rejects. No retry loop inside the terminal gate.

Ralph constraint loop

naac-ralph runs final boss, finds the reddest invariant, fixes one thing, and reruns until green or blocked.

SDK lesson

Every discovered exploit becomes a permanent constraint. sdk.eval must be peer to sdk.agent and sdk.flow.

Synthesis

GenUI <> SDK = schema-first surface contract

graph LR
  SDK[SDK actions + session schema] --> STREAM[LevEvent / PolyFrame stream]
  STREAM --> IR[LevUI IR / GenUI spec]
  IR --> HOST[AgentPing or host surface]
  HOST --> ACTION[Typed human/model action intent]
  ACTION --> POLY[SDK / PolyOp invocation]
  POLY --> RECEIPT[PolyReceipt + workstream graph]

The UI is a projection of SDK-declared actions and runtime events. It is not a second runtime.
