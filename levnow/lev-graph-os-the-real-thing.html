THE EPIPHANY

Graph isn't the source of truth. The world model is.

For four days we ran adversarial validation rounds against the graph kernel. Each round 'proved' the kernel. Each round was the kernel grading itself. False-positive loop.

Then the operator asked: what's the actual product? what does an agent or human invoke? where does this pay rent?

The honest answer was: nothing, nobody, and nowhere. The kernel was correct against itself. It had no consumer.

The interview that followed forced a re-grounding. The graph isn't truth. The graph is hygiene + virtual filesystem. Truth lives in the world model (and in code, for SDLC). The graph is what makes that truth queryable, projectable, and gate-able. Like Windows maintains indexes and macOS has special system folders, lev's .lev/ is a virtualized filesystem with explicit sync semantics. Graph-OS is the right name.

We built a graph that compiled. Now we're building a graph that's used.

GRAPH THEORY · IN ENGLISH

What's actually in the graph

There are 8 primitives in the kernel: entity, claim, evidence, proposal, gate, view, tick, intent. They never change. ADR-002 locked 4 node types for the runtime: agent, function, gate, subgraph. Cannot extend.

On top of that we have 11 rtype overlays — these are the cross-cutting facets that compose. lifecycle, execution, context, memory, skill, discourse, code, research, wiki, simulation, blackboard. The same entity can be tagged with multiple rtypes. A design document is discourse + wiki. A spec is lifecycle + execution. A flowmind YAML is execution. A wiki page is wiki + discourse + memory.

Over THAT we have buckets — typed content categories with their own schema and AGENT_SPEC. Buckets already exist underused at .lev/pm/<bucket>/_template/schema.json: proposal, design, spec, decision, plan, handoff, report. We're adding wiki-page, article, video, workstream, receipt.

Modifications are first-class. A proposal is the post-interview artifact that names: which N entities to patch, and which domain plugins make it real (the execution topology). A proposal that updates a website with a new header video would touch plugins/video AND plugins/sdlc simultaneously — that's why overlays matter.

DNA cascade is the propagation pattern: edit dna/core/<x> → compositor 4-phase fires → rtype overlays react → plugin owners activate → pentagon gates green or tmux teleport on red. Already shipped end-to-end.

ONE GRAPH · MANY EYES

APPENDIX

Decisions D1–D11 ratified in this interview

D1 · One kernel substrate

Not 5 substrates. Not file-tree-as-source. ONE kernel + N bucket schemas + 11 rtype overlays.

D2 · 4 ADR-002 node types are locked

agent / function / gate / subgraph. Cannot extend. DNA cascade is propagation pattern, not node type.

D3 · Beads is reference pattern only

Custom-content-types come from .lev/<bucket>/_template/schema.json. Build our own SDK. Beads stays as a study target.

D4 · Pentagon is SDLC-specific

Adversarial test of execution surfaces (CLI / MCP / SDK + spawn-agent-per-flow-branch). Not for wiki.

D5 · Wiki gates differ

Lint / orphan / health (autowiki materialization + discourse style). Different gate suite per bucket type.

D6 · Proposal in the new world

Post-interview artifact. Two parts: graph patch ops on N nodes + execution topology naming domain plugins.

D7 · Overlays compose multi-rtype proposals

A website update with new header video touches plugins/video AND plugins/sdlc simultaneously. That's why overlays matter.

D8 · Graph is hygiene, world model is truth

SDLC truth = code + graph (graph for hygiene). Other domains have their own truth substrates the graph references but does not own.

D9 · Graph-OS = OS file system semantics

.lev/ is a virtual filesystem with explicit sync policies. Code targets are user-declared hot paths (instant / lazy / reactive).

D10 · First slice = wiki + workstream lifecycle

Plan/act/verify loop preserved. Get out of markdown hell on the bucket types we already maintain by hand.

D11 · Autowiki + openbrain fusion

Karpathy materialization + Nat B Jones structured data. Best of both: originals immutable, LLM extracts structure not prose, projections regenerable. https://youtu.be/dxq7WtWxi44
