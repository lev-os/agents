LANE A · CONVENTION AUDIT

Repo Conventions After M9

What web-dx/packages/core/runtime/ looks like now, and the seam M10+ plans must fit

🧱

runtime/ is a flat kernel

Not a nested fractal. One xxxRuntime.js or xxxOwnership.js per domain, plus slices/xxxSlice.js factories. Do NOT create runtime/providers/ or runtime/hooks/ — the fractal is conceptual (slice + actions + compat = mini-provider), not directory.

🔗

utils.compose triad

import * as utils from './SameName' → export const composeX = (data, methods) => (payload) => {...} → useCallback(utils.composeX(data, methods), [deps]). Unit-test the helper directly with mock setters; integration-spy the compose call. Never re-test business logic through the component.

⚙️

createRuntimeXActions signature

createRuntimeXActions({ runtimeStore, ...refs }) => { actionName }. Guards with if (!runtimeStore?.getState || !runtimeStore?.setState) return {};. Local updateXSlice(reducer, actionType) closure calls runtimeStore.setState(next, false, { type: 'domain/actionName' }) — third arg is the devtools label.

🌐

DXL is weird — source-aware routing

Same DXL meta key routes to different legacy handlers depending on source.path (see normalizeDxlResponse.js:22-44). useSessionAction → DONOTUSE_broadcastInitAction; OverlayProvider → updateContent; DynamicOverlayComponent → invoke. Any M10+ that adds a source MUST update getLegacyHandlerForInitAction.

🧪

4-tier testing ladder

Unit (sibling X.test.js) → Integration (renderHook + RTL mount) → Golden trace (__tests__/ssot/*.golden.test.js) → Harness (agent-browser via pnpm ssot:dev:status -- --browser). Ownership migrations require all four tiers; pure runtime fns only need unit.

🏷️

Naming sunsets

Generic names only: legacyPageState, pageGraphState, runtimeState, runtimeHydrationState. Apollo names allowed only at existing API boundaries. DONOTUSE_* prefix for compat APIs awaiting rename. Sunset list closed: apolloState, __APOLLO_STATE__, getApolloClient, getDataFromTree, ApolloProvider, RouteContainer Apollo useQuery.
