1. Four-Tier JIT Execution Architecture

Every instruction token in FlowMind — {{var.path}}, [[scope:action]], lev://module — routes through a unified IR to the cheapest tier that can resolve it. The instruction graph (what) is stable; the execution topology (where + who) is chosen per session via host adapters.

Host adapters are hexagonal ports: CLI (argv/env/files), Browser (DOM + Agent Ping extension), Desktop (native controls), Remote (forwarded agent). Same instruction graph, different materialization. The WASM kernel ships at 297KB — instant load, zero network.

4. Bonsai-8B Deep Dive

PrismML's end-to-end 1-bit language model. Each weight is a single bit with one FP16 scale per 128 weights. 14.1× smaller than FP16. The only model to score 100% on boolean tasks.

Bonsai-8B is based on Qwen3 architecture with built-in <think> blocks. The model reasons internally before answering, adding 2-3s of latency but improving accuracy on harder tasks. With max_tokens=256 and think-stripping, it achieves 72% — tying phi4-mini.

The native GGUF Q1_0 format (1.15GB) requires PrismML's custom llama.cpp fork with 1-bit dequantization kernels. Standard Ollama cannot load Q1_0 — we tested the bartowski Q4_K_M standard quantization (5.76GB). Through the PrismML fork with Metal kernels, expect ~6× faster inference (~650ms).

9. Conclusions & Insights

Insight 1: Small Models Have Real Use Cases
SmolLM2-360M (130MB) achieves 100% on sentiment and comparison at 36ms. That's a browser-downloadable model that can reliably answer "is this positive or negative?" and "which of A/B is better?" — two of the most common gate predicates in content workflows.
Insight 2: Safety Is a Frontier-Only Task
Every model scores exactly 33% on safety (vulnerability detection). SQL injection, XSS, and input sanitization require reasoning about code semantics that sub-8B models cannot do. This validates the four-tier architecture — safety gates MUST route to T2.
Insight 3: Thinking Models Need Special Handling
Qwen3-based models (qwen3:4b, bonsai-8b) produce <think> blocks before answering. With max_tokens=10, the answer gets cut off (0% accuracy). With think-stripping and max_tokens=256, Bonsai achieves 72%. The thinking overhead adds ~2-3s but improves accuracy.
Insight 4: The Execution Topology Is Host-Independent
The instruction graph (what to resolve) is stable. The execution topology (where) changes per deployment: CLI uses Ollama, browser uses WebLLM + IndexedDB, Agent Ping ships the 297KB WASM kernel plus optional model download, remote agents forward the same plan with different credentials. One IR, many hosts.
Insight 5: Many Tiny Sims Can Parallelize
A 135M model at 50ms per inference can run 20 classification tasks in 1 second. Elixir-style process-per-agent supervision means you can fan-out cheap gate evaluations across many tiny models simultaneously — CDO-style consensus from diverse small models rather than one big model's opinion.
