ARCHITECTURE TAXONOMY

Compiled Declaration Architecture, Without Architecture Slop

One long-form super JSON for the family, the spine, the tag soup, and the session convergence.

April 2026 · GPT compare + repo sweep + CDO convergence

The session converged on one core answer: Lev does have a recurring compiler-shaped architecture, but it is not the universal ontology of the repo. The right framing is a **Compiled Declaration Architecture** applied to a **compiled-declaration spine**. That spine is strongest in FlowMind, Orchestration, Poly, and likely LevUI. Everything else should be classified honestly through planes, families, fit classes, and failure classes rather than being forced into the same mold.

CONVERGENCE

GPT and CDO landed in the same neighborhood

The overlap is high. The remaining delta is mostly about scope and naming discipline.

🤖

GPT synthesis

GPT got remarkably close: declaration -> compiler/lowering -> stable IR -> runtime/projection -> optional federation, four runtime planes, authored roots as truth, compiled view as frozen meaning, and runtime as proof-emitting state machine. The main overreach was treating the family slightly too broadly and using substrate language where the repo already reserves deeper substrate meanings.

🧠

CDO synthesis

CDO forced the edge cases: logger, LevEvent, projections, federation, YAML-as-projection, and root DNA scope. It converged on Compiled Declaration Architecture as the external family name, compiled-declaration spine as the internal name, and an optional DNA block instead of a new universal root doctrine.

✅ Settled

Use Compiled Declaration Architecture for the family. Use compiled-declaration spine for the scoped Lev-native application. Keep lens language only for governance and fit decisions.

How the session converged

From Poly/FlowMind intuition to a scoped architectural family with a contract sketch.

T2

Cross-repo sweep

Core, plugins, and specs were sampled. Strong in the FlowMind/Orchestration/Poly/Graph/UI spine. Partial and non-fits became explicit.

T3

Meta-substrate doctrine saved

The first doctrine note was written, but scope and naming were still too fuzzy.

T4

CDO taxonomy pass

Architecture vs lens vs substrate vs topology was tested against logger, LevEvent, federation, and the actual spine.

T5

Convergence

The family name settled on Compiled Declaration Architecture, scoped to the compiled-declaration spine, with lens retained as the governance guardrail.
