OPTION A

Separate Brands — Status Quo+

Two apps, shared backend, no unified story

ClawTok stays as the iOS content app. ClawBuddy stays as the web marketplace. They share kingly.cloud for inference but are otherwise independent products with separate brands, separate user bases, and separate App Store listings.

**community/kingly-assistant** (OpenClaw fork) powers an AI assistant inside ClawBuddy. graph-os.ai and graph.store are parked for future use.

This is the safest path. Ship both apps independently. Cross-promote later. No renaming, no architectural entanglement.

VERDICT

Option A Assessment

Strengths

Simplest execution. No renaming. Each app has clear identity. Ship faster with smaller scope. Two chances at App Store discovery. Independent failure modes.

Weaknesses

No unified brand story. Hard to cross-sell. 'Why are these related?' Leaves graph-os.ai on the table. No enterprise angle. Two marketing budgets. No network effects between apps.

OPTION B

Unified Under ClawBuddy

One brand, one app, one store — ClawTok becomes ClawBuddy iOS

Rename ClawTok to ClawBuddy. The iOS app becomes **ClawBuddy** — your AI companion that combines content discovery (the feed), the marketplace (browse/compare/buy agents), creator tools (generate, remix), and an AI assistant.

The web marketplace at frey-market becomes **graph.store** — the marketplace URL, powered by the same backend. ClawBuddy Store.

**"ClawBuddy" = the buddy.** It watches AI content with you, helps you find agents, assists with creation. One brand, one App Store listing, one story: *"Your AI companion for everything."*

graph-os.ai is parked or becomes the developer portal / about page. kingly.cloud stays as the inference layer, visible to power users as "Powered by kingly.cloud."

VERDICT

Option B Assessment

Strengths

One brand to market. 'ClawBuddy' has legs — buddy = companion = assistant. TikTok features make it sticky, marketplace makes it monetizable. One App Store listing with massive feature set. graph.store as the web URL is memorable.

Weaknesses

iOS app scope balloons — feed + store + assistant + creator tools. Harder to ship fast. Risk of 'trying to be everything.' No enterprise story. graph-os.ai still unused. The 'buddy' brand might not scale to enterprise.

OPTION C

graph-os.ai — The Full Platform Play

Enterprise white-label + consumer app + federated marketplace + open-source runtime

**graph-os.ai is THE platform.** It's what Lev becomes when it grows up. The operating system for AI agents — open-core, constraint-enforced, graph-native.

**graph.store** is the federated marketplace protocol. Anyone can run a graph.store node. Your flagship instance is the main marketplace. Enterprises run `company.graph.store` backed by the same protocol. The graph IS the marketplace — every agent is a node, every capability is an edge, compare = subgraph diff, discovery = graph walk.

**ClawBuddy** is the consumer face. Fun, social, accessible. Combines content + store + assistant. Built ON graph-os, not AS graph-os.

**kingly.cloud** is BOTH infrastructure (powers everything under the hood) AND a standalone product (developers buy API access, enterprises get private inference pools, white-label available).

The mesh node concept: run a node, serve inference, earn credits. `lev deploy --mesh` — agents self-register to the graph, inference routes through kingly.cloud, marketplace updates automatically.

**graph-os.ai = one of Lev's enterprise manifestations.** Lev is open source. graph-os.ai is the managed, SLA-backed, dashboarded version that enterprises pay for.

PRODUCT LINES

What graph-os Ships

For Consumers: ClawBuddy

Download the app. Watch AI content. Browse the store. Get an AI buddy that helps you find, compare, and use agents. Free tier with ads, premium with kingly.cloud credits.

For Developers: kingly.cloud + Lev

Open-source Lev runtime. kingly.cloud API for inference. Publish agents to graph.store. AgentPing for trust signals. lev deploy --mesh to join the network.

For Enterprise: graph-os.ai

Managed graph-os dashboard. Private graph.store nodes. Private kingly.cloud inference pools. Constraint enforcement. Audit trails. SLA. lev-forge CI/CD.

For Creators: ClawBuddy Studio

Generate content with any AI tool via kingly.cloud. Publish to the feed. Monetize via credits. Analytics dashboard. Collab tools. Cross-post to X/TikTok/YouTube.

KINGLY.CLOUD

kingly.cloud Triple-Dip

kingly.cloud becomes three things at once: infrastructure for your own apps (ClawBuddy uses it), standalone SaaS product (developers pay for API access), and enterprise white-label (companies run their own inference pool). That triple-dip is the actual business model.

VERDICT

Option C Assessment

Strengths

Category-defining positioning. Competes at platform level. Enterprise revenue path. Federated = network effects. 'graph-os' is unique and memorable. Every domain gets used. All existing code maps cleanly. Open-core model proven (GitLab, Supabase, PostHog).

Weaknesses

Most complex to execute. Enterprise sales cycle is long. The graph needs to actually work as described (not just internal plumbing). Federation is architecturally hard. Risk of over-positioning before the product proves itself. 'OS' overpromises.

Week 3-4

Ship Compare + AI Assistant

Compare tray in graph.store. Assistant bubble in ClawBuddy iOS. Both powered by kingly.cloud.

Week 5-6

AgentPing + Store tab in iOS

Trust signals on marketplace. Store tab inside ClawBuddy iOS app. Cross-sell begins.

Week 7-8

kingly.cloud standalone launch

Developer portal. API keys. Usage dashboard. Pricing tiers. OpenRouter competitor.

Month 3

graph-os.ai developer portal

Platform docs. 'What is graph-os?' page. Lev + kingly.cloud + AgentPing under one umbrella.

Month 4-5

Federation: graph.store protocol

Anyone can run a graph.store node. Agent registry protocol. Mesh inference routing.

Month 6

Enterprise: graph-os.ai managed

Dashboard. Private nodes. SLA. Compliance. First enterprise design partner.

RECOMMENDATION

The Play

Ship Option B now, grow into Option C. The rename is cheap. The architecture is the same code. The brand consolidation pays off immediately. As federation + enterprise features land, graph-os emerges organically from what you've already built.

**Day 1:** Rename ClawTok → ClawBuddy. Point graph.store → frey-market. Ship.

**Month 1:** ClawBuddy has feed + store + assistant. kingly.cloud powers generation. AgentPing provides trust.

**Month 3:** "What's graph-os?" becomes a real question. The answer: it's what powers all of this.

**Month 6:** Enterprise wants private graph.store nodes. graph-os.ai is the answer. Option C has arrived — not because you built it all at once, but because you grew into it.
