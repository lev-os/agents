MVP SPRINT

Claw Buddy MVP — 3–4 Weeks

Ship themes + compare + trailers first. AgentPing integration and agent-first/skills are post-MVP.

1

. Light/Dark + 3 Themes

Priority: P1 | Effort: 2-3 days | Score: 63

System-aware toggle + 3 named themes: Classic (gold/black Netflix), Midnight (deep blue/silver), Dawn (warm light). CSS custom properties already exist — extend with next-themes or custom provider.

Why first: Table stakes. Unlocks visual identity flexibility.

2

. Compare Feature

Priority: P1 | Effort: 1 week | Score: 70

Side-by-side comparison tray (up to 4 items). Feature matrix grid, pricing comparison, AI-generated summary, shareable link.

Why second: Drives purchase conversion. Essential marketplace UX.

3

. Trailers

Priority: P1 | Effort: 1-2 weeks | Score: 67

Demo videos + interactive previews showing what agents can do. Autoplay on hover, full-screen on click. Trailer upload in seller flow.

Why third: Show don't tell — differentiator over static listings.

4

. AgentPing + Lev Integration

Priority: P2 | Effort: 2-3 weeks | Score: 75 (highest)

Agent listing cards via AgentPing, real-time status, Lev platform hooks. Core value prop.

Post-MVP: Requires AgentPing API stabilization.

5

. Agent-First / Skills

Priority: P2 | Effort: 3-4 weeks | Score: 73

Agents self-register, manage listings via API. Skill discovery, verified badges. THE mission.

Post-MVP: Depends on skill system stabilization.

6

-7. /now Pages + Workshop

Priority: P3 | Effort: 2-3 weeks each | Scores: 66, 63

Auto-generated agent status pages via lev-now + Gemini avatars. Workshop/intake for auto-discovering capabilities.

Post-MVP: Nice-to-have.

MVP SPRINT

ClawTok MVP — 2–3 Weeks

Ship category view + profile redesign + podcasts. kingly.cloud inference is a separate epic.

1

. Discover: CLP Tier-1 Categories

Priority: P0 | Effort: 2-3 days | Score: 80 (highest overall)

Full-screen category grid as Discover entry point. 10 categories with icons, gradients, video counts. AICategory enum already exists.

Why first: Highest score. Clean taxonomy = machine-parseable = AI-friendly.

2

. Profile Redesign

Priority: P1 | Effort: 1 week | Score: 66

Split portfolio (view mode) from creator tools (creator mode). Horizontal scrolling quick-action bar. Overflow menu. Creator shortcut tray.

Why second: Fixes too busy problem. Clean separation of concerns.

3

. Music → Podcasts

Priority: P1 | Effort: 1-2 weeks | Score: 68

Extend Music module: Podcast model, podcast player with chapters/speed/sleep timer, AI podcast generators (NotebookLM, ElevenLabs, Play.ht).

Why third: Content diversity. AI podcasts is a growing niche.

4

. kingly.cloud Inference Router

Priority: P3 | Effort: Months | Score: 66 (GO with caveats)

Cross-cutting infra: multi-provider LLM routing, white-label API, credit billing, rate limiting. Separate epic with 20-agent CDO swarm.

Deep research running now → docs/_inbox/

CRITICAL GAPS

What Must Be Fixed Before MVP

Zero Tests — Both Projects

Neither project has any test infrastructure. Claw Buddy needs Playwright + Vitest. ClawTok needs XCTest + UI tests. This is the #1 blocker for production confidence.

ClawTok Backend Not Built

A 57K-word backend plan exists (Go + Fiber, PostgreSQL, Redis, R2, Temporal.io) but zero implementation. The entire iOS app runs on mock data. Supabase schema is ready (8 migrations) but needs a Go API server.

Dashboard Runner Port Mismatches

Frey Market: dashboards.yaml says port 3100, detected-projects.yaml says 3115. ClawTok: dashboards.yaml targets iPhone 17 Pro, detected-projects targets iPhone 16 Pro. Both need reconciliation.

No CI/CD Pipeline

Neither project has GitHub Actions, Vercel config, or Xcode Cloud setup. Need: GH Actions → Vercel for ClawBuddy, Xcode Cloud → TestFlight for ClawTok.

CROSS-CUTTING EPIC

kingly.cloud Inference Router

The inference router is the infrastructure backbone that serves both apps. Multi-provider LLM middleware with white-label capability, credit-based billing, and intelligent routing.

**Deep research agent running now** — writing to `docs/_inbox/kingly-cloud-inference-router-research.md`. Architecture includes: 20-agent dynamic CDO swarm, codex-autoresearch loops, levloop tick scheduling, dynamic skill lookup per turn, web + code research every tick.

**Competitive landscape:** OpenRouter, LiteLLM, Portkey, Martian, Unify.ai, Helicone. kingly.cloud differentiates via Lev ecosystem integration, agent-native design, and integrated credit system.
