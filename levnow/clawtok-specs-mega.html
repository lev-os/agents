SPEC 1 — P0 MVP

Discover: Category Landing Pages

The horizontal scroller shows 8 of 12 AICategory cases as small tiles. Tapping sets search text — no dedicated category page, no visual hierarchy.

DiscoverView.swift is 486 lines with a category scroller that only shows `.prefix(8)` of 12 AICategory cases. Tapping a category just sets `searchText` to the rawValue and filters the trending grid. There is no full-screen category landing page. For a platform built around AI content taxonomy, this is the critical gap. The `AICategory` enum already exists in Models.swift with 12 cases, each having an `icon` (SF Symbol), `color`, and `rawValue`. The data model is ready; the UI is not.

**User feedback:** "needs a top level category view (CLP tier 1 (ai will understand this))" — the taxonomy must be machine-parseable for AI search indexing.

**CategoryGridView.swift** — Full-screen 2-column grid replacing the horizontal scroller as primary Discover entry. Each cell: SF Symbol icon, `LinearGradient` from `category.color.opacity(0.6)` to `0.2`, name, video count badge, trending flame indicator (>10% growth in 7d). Uses `LazyVGrid` with 2 `GridItem(.flexible(), spacing: ClawTokSpacing.md)`. All 12 cases displayed — remove `.prefix(8)` limit.

**CategoryFeedView.swift** — Filtered vertical scroll of `VideoCardView` instances where `video.category == selectedCategory`. Sticky category bar at top: horizontal scroll of all categories, selected highlighted with brand gradient underline. Pull-to-refresh placeholder. Infinite scroll stub.

**StickyCategoryBar.swift** — Reusable horizontal category selector. Active category gets gradient underline using `ClawTokColors.categoryGradient(for:)`.

**Decision:** Use a `DisplayCategory` enum mapping to `AICategory` for MVP (Option A). No model changes needed. Avoids video data migration.

SPEC 2 — P1 MVP

Profile: View/Creator Mode Split

ProfileView.swift is 698 lines combining portfolio viewing with creator tools. "There's 2 mental models here."

Profile is too busy (mixing view portfolio + creator tools). Good for edit mode/creator mode though. The buttons can scroll horiz or we need a ... next to the tabs for all the quick actions. There's 2 mental models here, creator tools plus my profile, creator tools should have a shortcut tray somewhere.

**View Mode (default):** Clean portfolio. Avatar + activity ring, Following/Followers/Likes stats, bio + link, Edit/Share/Analytics buttons, divider, content tabs (Videos / AI Music / Liked / Bookmarks) with "..." overflow menu, content grid. Creator tools section completely hidden. This is what visitors see on UserProfileView (minus edit, replaced with Follow + Message).

**Creator Mode (toggle):** Everything in View mode PLUS horizontal scrolling quick-action bar between action buttons and content tabs. Contains: +Create, Live, AI Tools, Playlists, Creator Hub, AI Music, Credits, Drafts, Upload History. Uses `ScrollView(.horizontal)` with capsule pills (same style as current `creatorToolPill`). No paging, no snapping. The 2x3 `LazyVGrid` (lines 212-229) is replaced entirely.

**Floating Shortcut Tray (post-MVP):** Small "+" FAB in bottom-right (above tab bar). Taps to expand radial fan of 3 most-used tools (Create, AI Tools, Live). Appears in BOTH modes. Uses `matchedGeometryEffect`. Can be disabled in Settings.

**Mode toggle (Option C recommended):** Single icon button in toolbar trailing area, next to QR and Settings. Eye icon (`eye.fill`) for view mode, wrench icon (`wrench.and.screwdriver.fill`) for creator mode. Persists via `@AppStorage("profileMode")`. UserProfileView ALWAYS shows view mode — no toggle.

**Animation:** Creator tools row slides up/down with `.transition(.move(edge: .top).combined(with: .opacity))`. Toggle icon rotates 180 degrees. Spring: `response: 0.4, dampingFraction: 0.8`.

SPEC 3 — P2 POST-MVP

Music Module: Podcast Support

The Music module handles tracks but has no long-form audio. AI podcasts (NotebookLM, ElevenLabs) are exploding — ClawTok needs this vertical.

Podcasts are structurally different from tracks: episodes, hosts, transcripts, chapters, different playback UX (speed control, sleep timer, chapter navigation). Bolting podcast fields onto Track violates separation of concerns. New `Podcast`, `Episode`, `Chapter` models alongside existing `Track`.

**7 AI podcast tools:** NotebookLM, ElevenLabs, Play.ht, Descript, Podcastle, Wondercraft, Other. **8 categories:** AI News, Tech Deep Dive, Tutorials, Interviews, Commentary, Research Papers, Creative/Fiction, AI Business.

**MusicHub integration:** Add `MusicHubTab` enum (Music | Podcasts). When Podcasts selected, show PodcastDiscoverySection with featured horizontal scroll, category chips, "NotebookLM Originals" promoted section.

**Player:** Full-screen with cover art, progress bar with chapter markers, 15s skip, chapter navigation, speed control (0.5x-2x), sleep timer (15m/30m/45m/1h/end), transcript panel synced to playback.

**Upload:** 6-step flow: audio file -> metadata -> transcript (paste/auto) -> chapters (manual timestamps) -> cover art -> review + publish.

SPEC 4 — P3 SEPARATE EPIC

kingly.cloud Inference + Lev LLM Router

10 AI tool screens are static mockups. No inference happens. Building direct provider integrations from iOS is wrong — API key leaks, no billing, no failover.

kingly.cloud is a unified inference gateway serving ClawTok, ClawBuddy, and white-label apps. Single endpoint (`POST /v1/inference`) with async job queue, credit reservation/settlement, and multi-provider routing. The Lev LLM Middleware Router handles model selection, provider health checks, rate limiting, cost optimization, request transformation, and response normalization.

**Major prior art found:** `plugins/claw-router/` is already a functional OpenAI-compatible proxy with `/v1/chat/completions`, model aliasing (auto/eco/premium/free), routing strategies (balanced/cheap/quality/free), x402 crypto payments, and streaming. **This is not greenfield — it's a refactor + multi-tenant extension.**

**5 deep research agents running now** analyzing: competitive APIs, provider landscape, credit systems, gateway architecture, and claw-router internals. Results landing in `docs/_inbox/`.

**Credit flow:** Pre-flight check (402 if insufficient) -> Reserve credits on queue -> Consume on completion -> Release on failure. Append-only ledger in Supabase: purchase, consume, refund, gift, promo transaction types.

**iOS integration:** `InferenceService.swift` in `Core/Services/`. Each AI tool screen calls `.submit(toolType:input:)` which POSTs to `/v1/inference`. Progress view while processing (poll or push notification). Credit balance in toolbar, updated after each job.

**White-label:** Each tenant has `app_id` in auth token. Per-tenant config: allowed tool types, credit pricing, provider preferences, rate limits. SDKs: `KinglyCloudSDK` (Swift), `@kingly/cloud` (npm).
