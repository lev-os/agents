ARCHITECTURE

The Spine Pattern

SDK-first 3-layer architecture extracted from AppTestr — 66 RPC methods across 10 domains

🧱

SDK-First Layers

sdk/ — business logic (composable method routers) wrappers/ — adapters (CLI, MCP, HTTP, TypeScript SDK) driver/ — platform native code (Swift AX, browser, storage)

Wrappers MUST NOT own business logic. Enforced by lint gates.

🔀

Composable Routers

Domain-specific routers implement canHandle(method) + handle(method, params). Central createMethodRouter() chains them. Adding new domains doesn't touch dispatch.

⚙️

Daemon Lifecycle

Unix socket, JSON-RPC, socket-locked singleton. CLI never runs business logic directly. The daemon owns all runtime state: sessions, recordings, dev lifecycle.

👁️

Observe-Act-Verify

Single-step discipline. Multi-action plans without verification are forbidden. Maps directly to the eval flow: produce → judge → compare → keep_or_discard.

🚧

Layer Enforcement

500/700 LOC gates, layer import gates, and tsc --noEmit are hard CI gates. Not suggestions — enforcement.

🗂️

7 Domain Routers

LANDSCAPE

Browser Automation + AgentFS

Three browser strategies and a filesystem sandbox that makes constraint-checking inescapable

🧪

dev-browser

Self-documenting CLI — --help IS the LLM guide. CLI → daemon (persistent page state) → QuickJS WASM sandbox → Playwright. Sandboxed scripts can't touch host filesystem.

🤖

agent-browser

80+ commands. Rust CLI → Node.js daemon → Playwright. Ref-based workflow (snapshot → refs → interact). ~200-400 tokens per interaction.

🏆

chrome-devtools-axi

AXI-principled wrapper around CDP. Combined operations (fill @uid text --submit fills, submits, waits, returns snapshot). Lowest cost, highest success rate.

📁

AgentFS Sandbox

Single SQLite file = VFS + KV store + tool call audit trail. Overlay CoW: fork state → run constraints → commit or discard. Agents can't bypass filesystem-level isolation.

TICK ANATOMY

One Tick = One Intent Bucket

Six steps per tick, strictly scoped — no drift allowed

Step 1 · Inventory

Count template rows for this intent. Count graph nodes. Check runtime regex coverage. Output: inventory section in the tick log.

Step 2 · Verify Templates

For each row: Is the intent label correct? Is the text real FAA phraseology? Are slots correct and complete? Does _source have section + page + docling_ref? Flag invented templates for removal.

Step 3 · Add Readbacks

Add expected_readback field using readback format overlay, AIM 4-4-7 requirements, and partner extractor patterns. Readback slots must match template slots.

Step 4 · Verify Graph Alignment

Do graph nodes exist? Do params match template slots? Does section_ref match _source.section? Flag all misalignments.

Step 5 · Extract Runtime Rules

Check intent_policy_overlay for regex coverage. Mine partner extractors for patterns. Log recommendations — DO NOT modify the overlay.

Step 6 · Write Changes

Update base.json for this intent ONLY. Allowed: fix _source, add readbacks, fix slots, fix labels, add needs_review. Forbidden: add/remove rows, edit graph files, edit modules.
