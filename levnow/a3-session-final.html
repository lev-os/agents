The Four Nouns (Overlay Model)

Plugins project influence via overlays. Tasks are fractal mini-roots.

Ontology

What CAN exist. System invariants, entity types, lifecycle domains.

Lives in: dna/graph.yaml + dna/entities.yaml Scope: System-wide (frozen) Authority: Canonical invariant

DNA (was: Definition)

What IS this specific thing. Intent, acceptance, local constraints.

Lives in: .lev/tasks/<id>/dna.yaml Scope: Per-task (authored) Authority: User intent

Renamed from 'definition' to 'dna' per other agent's canonical lifecycle_dimensions.

Execution

HOW it gets carried out. Topology, validation chain, receipts.

Lives in: .lev/tasks/<id>/execution.yaml Scope: Per-task (compiled) Authority: Compiled from DNA + overlays

Standards + architecture are execution subdomains, not siblings.

Projection

How it APPEARS. Lossy scoped views of graph state.

Lives in: plugins/projections/<name>/ Scope: Multiple per entity Authority: Derived
