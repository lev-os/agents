Events Daemon Plan

Manual first was the right call

Current event truth

.lev/events.jsonl is a project-local projection used by daemon and exec paths. Other logs exist today: ~/.local/share/lev/events.jsonl, ~/.local/share/lev/daemon/state.jsonl, and .lev/agentfs/exec/events.jsonl. event-dispatch reads a JSONL source; it does not own ingestion.

Daemon target

For the MVP hot path, keep one ordered project event stream for replay simplicity, then treat specialized logs as evidence/projection stores. The daemon should automate the manual rail: observe, project, dispatch through SDK/exec, and emit one linked receipt.
