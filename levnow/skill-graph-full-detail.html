Shape — Every Feature In Full

11 Lev skills + 6 external features + 7 roles

### Lev Skills in Shape (11 total)

**`interview`** — Wizard-mode questioning skill for structured decision-making. Uses framework library (SCAMPER, First Principles, Systems Thinking). Guides through complex decisions one question at a time.

**`cdo`** — Adaptive multi-agent deliberation for cdo/think/deep/debug/parliament workflows. Fractal context loop (CONTEXT→PLAN→ACT→VERIFY) at every level. Mandatory Turn 0 context gathering (skills, codebase, web). Convergence requires evidence, not just agreement.

**`prior-art`** — Evidence-first provenance report from `cass`, repo docs, `.lev`, `~/.agents/diagrams`, `qmd`, `lev get`, or `rg`.

**`cass`** — Search wrapper around the external `cass` CLI for coding-agent session history. Uses `cass` in machine-readable mode (`--json` or `--robot`).

**`cass-memory`** — Procedural memory wrapper around the external `cm` CLI (CASS Memory System). Three-layer cognitive loop.

**`lev-research`** — When any research, search, or information gathering is needed.

**`sherlock`** — Sherlock Bytes: Deep Research Intelligence System.

**`codebase-archaeology`** — Systematically explore unfamiliar codebases to build working mental models. Use when onboarding to new project, "what does this do", or understanding legacy code.

**`skill-discovery`** — Local skills runtime lookup over the canonical Leviathan skills inventory. Uses `~/.agents/lev-skills.sh` + `~/.agents/skills-inventory.jsonl` as the source of truth.

**`search`** — Unified Search Router.

**`brainstorming`** — (deprecated, superseded by `interview`)

ooo-04 · Ouroboros · 3-stage eval gate

Full name: 3-stage eval gate (Mechanical → Semantic → Multi-Model)

Status: implemented

The 3 stages (verbatim):

  • Mechanical — does it compile? do tests pass? exit code 0?
  • Semantic — does the output match the intended meaning? (metric-based)
  • Multi-Model — do multiple independent LLMs agree the output is good?

Notes (verbatim): "EvalStrategy has 4 kinds: gate-based (mechanical), metric-based (semantic), llm-as-judge, adversarial. Eval harness flow implements the gating."

The powerful sequencing: Mechanical is cheap, fails fast, catches 80% of broken output. Semantic is medium cost. Multi-Model is expensive — only runs if Mechanical + Semantic both pass.

What to steal: the naming. "Mechanical/Semantic/Multi-Model" is clearer than "gate/metric/judge."

Lev equivalent: core/orchestration/src/eval/types.ts

**Ambiguity scoring (OMX omx-02):**

- Weighted dimensions: **intent 30% + outcome 25% + scope 20% + constraints 15% + success 10%** - Depth profiles: **quick ≤0.30, standard ≤0.20, deep ≤0.15**

**Ambiguity scoring (Ouroboros ooo-01):**

- Ambiguity gate threshold: **0.2** - Convergence threshold: **0.95** - Max generations: **30** - Cost tiers: **3** (PAL Router)

**Drift measurement (Ouroboros ooo-10):**

- Greenfield: `goal 50% + constraint 30% + ontology 20%` - Brownfield: `goal 40%, constraint 30%, success criteria 30%, context 15%`

**PAL Router tiers (Ouroboros ooo-07):**

- **1x / 10x / 30x** cost labels — triage/work/review - Mapped to haiku/sonnet/opus in lev-ralph

**3-stage eval gate (Ouroboros ooo-04):**

- **Mechanical** (compile, exit code) → **Semantic** (metric-based match) → **Multi-Model** (LLM consensus) - Run in order, cheapest first. Multi-Model only if Mechanical + Semantic both pass.

**Convergence / evolutionary loop (Ouroboros ooo-05):**

- Stop when ontology similarity >= **0.95** - Formula: `name_overlap + type_match + exact_match` - HOLD state = collect more samples before deciding

### The one-sentence powerful takeaways

1. *"Stop interviewing when ambiguity < 0.2"* (Ouroboros) 2. *"Stop evolving when ontology similarity > 0.95"* (Ouroboros) 3. *"Weight dimensions: intent 30% + outcome 25% + scope 20% + constraints 15% + success 10%"* (OMX) 4. *"Cost tiers: 1x triage / 10x work / 30x review"* (Ouroboros) 5. *"Gate stages: Mechanical → Semantic → Multi-Model"* (Ouroboros) 6. *"Drift = goal 50% + constraint 30% + ontology 20%"* (Ouroboros) 7. *"Auto-nudge when worker stalls"* (OMX) 8. *"Commit ledger shared across team"* (OMX) 9. *"Doctor says what's wrong in plain language"* (Hermes) 10. *"Onboarding is ceremony, not README"* (OpenClaw)
