The AgentPing unification vision

Three surfaces you already have. One scanner output. Merge them, and component review (400+ migrations) becomes visual triage inside the same interface.

Same data, three surfaces

The scanner at core/build/src/scanner.ts is already the discovery layer. It reads .lev/config.yaml projects: and writes .lev/detected-projects.yaml. A0 is consuming it for the tmux harness. A4 is pulling it into content dashboards. I'm rendering it in dashboard-manager-ui.

When we get to component review: each of the 400+ migrated @kingly/ui components gets a card in AgentPing. Click → GenUI renders the component with variants → you approve/reject → lifecycle transitions. The dashboard IS the review tool.

The detected→ready flow (P1) is the foundation: websocket endpoint in dashboard-manager-server, Setup button in dashboard-manager-ui, spawns lev exec, streams progress, mutates lifecycle on success. Same pattern extends to component review later.

Open architectural question

Websocket dispatch substrate

You asked: clawhip or FlowMind for the detected→ready websocket dispatch?

Option A · FlowMind

In-process TypeScript

core/flowmind/src/session.ts (409 LOC) already has session lifecycle: idle → running → paused → completed → failed. Persists to XDG. Ships with Lev. Direct import into dashboard-manager-server. WebSocket wraps session events.

Best for: the dispatch itself. Lifecycle mutation, lev exec invocation, progress streaming, state persistence.

Option B · clawhip

Rust tmux harness

MIT, 34K LOC. Already vendored by A0 for the tmux-control-plane P0. Handles tmux session spawning, lifecycle events, content hashing. Not TypeScript — would need FFI or subprocess.

Best for: the terminal side — if you want component review to spawn agents in tmux panes.

My recommendation: FlowMind for the websocket dispatch (it's the right tool for session state mutation), clawhip for anything that needs to spawn a terminal agent. They're not competing — they stack. FlowMind owns the session graph, clawhip owns the pane.
